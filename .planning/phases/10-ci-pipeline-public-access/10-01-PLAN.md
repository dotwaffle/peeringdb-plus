Create `.github/workflows/ci.yml` with the following structure:

**Triggers:** `on: pull_request` and `on: push: branches: [main]`

**Concurrency:** Cancel in-progress runs for the same PR/branch. Use `concurrency: group: ${{ github.workflow }}-${{ github.head_ref || github.ref }}` with `cancel-in-progress: true`.

**Four jobs, all running on `ubuntu-latest`:**

**1. `lint` job:**
- Checkout code
- Set up Go using `actions/setup-go@v5` with `go-version: stable` (per locked decision: latest stable only, NOT from go.mod, NOT matrix)
- Install golangci-lint: `go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest`
- Run `golangci-lint run` (uses .golangci.yml from Phase 7 at repo root)
- **Generate drift detection:** Run `go generate ./ent ./schema` then `git diff --exit-code` to catch uncommitted code generation changes. The generate directives are in `ent/generate.go` (`//go:generate go run -mod=mod entc.go`) and `schema/generate.go` (`//go:generate go run ../cmd/pdb-schema-generate/main.go peeringdb.json ../ent/schema`). If `git diff --exit-code` fails, the job fails with a clear message: "Generated code is out of date. Run 'go generate ./ent ./schema' and commit the changes."

**2. `test` job:**
- Checkout code
- Set up Go with `go-version: stable`
- Run `CGO_ENABLED=1 go test -race -coverprofile=coverage.out ./...` (per CI-02: race detector requires CGo even though production uses modernc.org/sqlite)
- Run `go tool cover -func=coverage.out` and capture output to `coverage.txt`
- Run `.github/scripts/coverage-comment.sh` to post coverage as PR comment (only on pull_request events, skip on push to main)
- Use `permissions: pull-requests: write` on this job so `gh api` can post comments

**3. `build` job:**
- Checkout code
- Set up Go with `go-version: stable`
- Run `go build ./...` to verify compilation
- No artifact upload (per locked decision: deferred)

**4. `govulncheck` job:**
- Checkout code
- Set up Go with `go-version: stable`
- Install govulncheck: `go install golang.org/x/vuln/cmd/govulncheck@latest`
- Run `govulncheck ./...` (blocks on called vulns by default, informational for unused — this is govulncheck's default behavior, no flags needed)

All four jobs run in parallel (no `needs:` dependencies between them).

Each job should cache Go modules via `actions/setup-go@v5` which handles caching automatically when `go-version` is specified.

Use YAML anchors or keep explicit for readability. Prefer explicit (no anchors) for CI files since they're easier to understand.

  <action>
Create `.github/scripts/coverage-comment.sh` as a Bash script that:

1. Reads `coverage.txt` (output of `go tool cover -func=coverage.out`) from the current directory
2. Extracts the total coverage percentage from the last line (format: `total:  (statements)  XX.X%`)
3. Formats a PR comment body in Markdown:
   ```
   ## Test Coverage

   **Total: XX.X%**

   <details>
   <summary>Per-package breakdown</summary>

   ```
   [full go tool cover -func output]
   ```

   </details>
   ```
4. Posts the comment to the PR using `gh api`:
   - Uses `GITHUB_TOKEN` (available automatically in Actions via `${{ github.token }}`)
   - Uses `$GITHUB_REPOSITORY` and `$PR_NUMBER` environment variables
   - The workflow will set `PR_NUMBER` from `${{ github.event.pull_request.number }}`
   - API call: `gh api repos/${GITHUB_REPOSITORY}/issues/${PR_NUMBER}/comments -f body="$COMMENT_BODY"`
   - Before posting, search for existing coverage comments by this bot and update instead of creating duplicates. Use `gh api repos/${GITHUB_REPOSITORY}/issues/${PR_NUMBER}/comments` to list comments, find one containing "## Test Coverage" from `github-actions[bot]`, and if found use PATCH to update it, otherwise POST to create new.

The script should:
- Be executable (chmod +x)
- Exit 0 even if comment posting fails (coverage reporting is informational, should not fail the build)
- Exit early with a message if `PR_NUMBER` is empty (push to main, not a PR)
- Print the total coverage to stdout regardless (so it appears in the job log)

Per locked decision: no external services (no Codecov, no Coveralls). This is a self-contained shell script using only `gh` CLI.
