# Plan 19-01: Prometheus Export & Grafana Dashboard

## Objective

Enable Prometheus metrics export via the OTel autoexport prometheus reader and create a comprehensive Grafana dashboard JSON covering sync health, HTTP RED metrics, per-type sync detail, Go runtime, and business metrics. The dashboard must be portable (datasource template variables, no hardcoded UIDs) and include documentation text panels.

## Context

- @internal/otel/provider.go - OTel setup using autoexport (already supports prometheus via OTEL_METRICS_EXPORTER=prometheus)
- @internal/otel/metrics.go - Custom sync metrics (freshness, duration, operations, per-type)
- @fly.toml - Fly.io deployment config (needs [metrics] section for Prometheus scraping)
- @go.mod - Already has prometheus dependencies as indirect via autoexport

## Tasks

### Task 1: Enable Prometheus metrics endpoint
type="auto"

**Do:**
1. Add `[metrics]` section to fly.toml pointing Fly.io Prometheus scraper at port 9091 path /metrics
2. Add `OTEL_METRICS_EXPORTER=prometheus` and `OTEL_EXPORTER_PROMETHEUS_PORT=9091` to fly.toml [env]
3. Write a test that verifies Setup() with OTEL_METRICS_EXPORTER=prometheus creates a functioning metrics endpoint on the configured port

**Done when:**
- fly.toml has [metrics] section and prometheus env vars
- Test confirms prometheus exporter initializes without error

### Task 2: Create Grafana dashboard JSON with all rows
type="auto"

**Do:**
1. Create deploy/grafana/dashboards/peeringdb-plus.json with a comprehensive Grafana dashboard containing:
   - Template variable for Prometheus datasource (no hardcoded UIDs)
   - Row 1: Sync Health - freshness gauge (green <1h, yellow <2h, red >2h), sync duration histogram, success/failure rate, fallback events
   - Row 2: HTTP RED Metrics - request rate by route, error rate (4xx/5xx), latency percentiles (p50/p95/p99), active requests
   - Row 3: Per-Type Sync Detail - duration by PeeringDB type, object counts, delete counts, fetch/upsert errors
   - Row 4: Go Runtime - goroutines, heap memory, allocation rate, GC goal
   - Row 5: Business Metrics - object counts per PeeringDB type, region breakdown
   - Documentation text panel in each row explaining metrics and troubleshooting
2. Create deploy/grafana/provisioning/dashboards/default.yaml for Grafana dashboard provisioning

**Done when:**
- deploy/grafana/dashboards/peeringdb-plus.json exists with all 5 rows and doc panels
- deploy/grafana/provisioning/dashboards/default.yaml exists
- Dashboard uses datasource template variable `${DS_PROMETHEUS}` throughout
- Each row has a documentation text panel

### Task 3: Verify dashboard JSON validity and metric references
type="auto"

**Do:**
1. Write a Go test that:
   - Parses the dashboard JSON as valid JSON
   - Verifies all required rows exist (sync health, HTTP RED, per-type, runtime, business)
   - Verifies datasource template variable exists (not hardcoded UIDs)
   - Verifies each row has a text panel (documentation)
   - Verifies key metric names in queries match actual OTel metric names (pdbplus.sync.*, http.server.*)
2. Run the test

**Done when:**
- Test passes confirming dashboard JSON structure and metric references
