  <action>
Two-pronged approach: measurement-level scoping via -coverpkg, and report-level exclusion via octocov.

1. Update `.octocov.yml` to add `coverage.exclude` patterns:

```yaml
coverage:
  paths:
    - coverage.out
  exclude:
    - 'ent/**/*.go'
    - 'gen/**/*.go'
    - 'graph/generated.go'
    - '**/*_templ.go'
comment:
  deletePrevious: true
```

IMPORTANT: Do NOT exclude `graph/schema.resolvers.go` or `graph/custom.resolvers.go` -- those contain ~500 lines of hand-written resolver code that gqlgen preserves during regeneration. Only `graph/generated.go` (57K lines, pure codegen) should be excluded.

2. Update `.github/workflows/ci.yml` line 63 (the test command) to scope coverage via -coverpkg. Replace:

```yaml
- name: Run tests with race detector
  run: CGO_ENABLED=1 go test -race -coverprofile=coverage.out ./...
```

With:

```yaml
- name: Run tests with race detector
  run: |
    COVERPKG=$(go list ./... | grep -vE '/(ent|gen)/' | tr '\n' ',' | sed 's/,$//')
    CGO_ENABLED=1 go test -race -coverprofile=coverage.out -coverpkg="${COVERPKG}" ./...
```

The `grep -vE '/(ent|gen)/'` removes all ent/ subpackages (21 packages) and gen/ subpackages (2 packages) from the coverage denominator. The top-level `ent` package is also excluded since `go list` returns `github.com/dotwaffle/peeringdb-plus/ent` which matches `/ent/` via the path separator.

Wait -- the grep pattern `/(ent|gen)/` requires a trailing slash. The top-level `ent` package path ends with `/ent` (no trailing slash). Let me fix the pattern to also catch `ent$`:

```yaml
COVERPKG=$(go list ./... | grep -vE '/ent(/|$)|/gen(/|$)' | tr '\n' ',' | sed 's/,$//')
```

This ensures both `github.com/dotwaffle/peeringdb-plus/ent` and `github.com/dotwaffle/peeringdb-plus/ent/campus` are excluded. Same for gen.

The `graph` and `schema` packages are NOT excluded from -coverpkg because they contain hand-written code. Only octocov file-level excludes handle `graph/generated.go` and `*_templ.go`.
