<objective>
Build the foundational infrastructure for the PeeringDB compatibility layer: type registry, generic filter parser, entity serializers, and response envelope.

Purpose: These components are the core building blocks that all HTTP handlers will depend on. The type registry provides a generic dispatch mechanism (avoiding 13-way switch statements), the filter parser translates Django-style query parameters to ent predicates, serializers map ent entities to PeeringDB's exact JSON format, and the response envelope produces PeeringDB-identical output.

Output: `internal/pdbcompat/` package with registry.go, filter.go, filter_test.go, serializer.go, serializer_test.go, response.go
</objective>

    Implement:
    - `parseFieldOp(key string) (field, op string)` -- splits on last "__" (e.g., "name__contains" -> "name", "contains")
    - `ParseFilters(params url.Values, fields map[string]FieldType) ([]func(*sql.Selector), error)` -- iterates params, skips reserved, validates field exists (silently ignores unknown per D-20), builds predicates
    - `buildPredicate(field, op, value string, ft FieldType) (func(*sql.Selector), error)` -- maps operator to sql.Field* function:
      - "" (exact): sql.FieldEqualFold for strings (case-insensitive per D-10), sql.FieldEQ for others
      - "contains": sql.FieldContainsFold (case-insensitive per D-10)
      - "startswith": sql.FieldHasPrefix (use FieldContainsFold pattern -- actually use raw LIKE with prefix% for case-insensitive)
      - "in": sql.FieldIn with proper type conversion (strings stay strings, ints converted per Pitfall 5)
      - "lt", "gt", "lte", "gte": sql.FieldLT, sql.FieldGT, sql.FieldLTE, sql.FieldGTE
    - `convertValue(s string, ft FieldType) (any, error)` -- converts string to typed value based on FieldType

    For case-insensitive startswith per D-10: since sql.FieldHasPrefix is case-sensitive, implement a custom predicate using LIKE with proper escaping: `func(s *sql.Selector) { s.Where(sql.Like(field, value+"%")) }`. SQLite LIKE is case-insensitive for ASCII by default.
