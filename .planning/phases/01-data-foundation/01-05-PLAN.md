    **Architecture:** Pattern-matching parser (not full AST) per research Pitfall 8 recommendation. Django model/serializer definitions follow predictable patterns that regex can handle reliably.

    **JSON Schema Output Format (D-13, D-14, D-15):**
    ```json
    {
      "version": "1.0",
      "extracted_at": "2026-03-22T12:00:00Z",
      "source_path": "/path/to/peeringdb",
      "object_types": {
        "org": {
          "model_name": "Organization",
          "api_path": "org",
          "base_classes": ["HandleRefModel", "AddressModel"],
          "fields": {
            "name": {
              "type": "string",
              "max_length": 255,
              "required": true,
              "unique": true,
              "read_only": false,
              "deprecated": false,
              "help_text": "Organization name",
              "default": null
            },
            "org_id": {
              "type": "integer",
              "required": false,
              "references": "org",
              "read_only": false,
              "deprecated": false,
              "help_text": "FK to organization"
            }
          },
          "computed_fields": ["net_count", "fac_count"],
          "relationships": {
            "networks": {
              "target": "net",
              "type": "one_to_many",
              "field": "org_id"
            }
          }
        }
      }
    }
    ```

    **Python Source Files to Parse:**
    The tool reads from the PeeringDB repo path (argument, per D-12):
    - `{repo}/peeringdb_server/serializers.py` -- field definitions, read_only, validators
    - `{repo}/../django-peeringdb/src/django_peeringdb/models/abstract.py` -- base model fields
    - `{repo}/../django-peeringdb/src/django_peeringdb/models/concrete.py` -- FK relationships
    - `{repo}/../django-peeringdb/src/django_peeringdb/const.py` -- choice values

    **Pattern Matching Strategy:**
    Use regex patterns to extract:
    1. `class {Name}Serializer` -- identifies object types
    2. `class Meta:` + `model = {Model}` -- maps serializer to model
    3. `fields = [...]` or `fields = "__all__"` -- field lists
    4. `{field_name} = serializers.{FieldType}(...)` -- field definitions from serializers
    5. `{field_name} = models.CharField(max_length=N, ...)` -- field definitions from models
    6. `ForeignKey({Model}, ...)` -- FK relationships
    7. `read_only_fields = [...]` -- read-only annotation
    8. `{CONST_NAME} = [(...), ...]` -- choice constants

    **Program structure:**
    ```go
    func main() {
        if len(os.Args) < 2 {
            log.Fatal("usage: pdb-schema-extract <peeringdb-repo-path> [--validate]")
        }
        repoPath := os.Args[1]
        validate := slices.Contains(os.Args[2:], "--validate")

        schema, err := extractSchema(repoPath)
        // ...

        if validate {
            // D-16: validate against live API
            if err := validateAgainstAPI(schema); err != nil {
                log.Fatalf("validation failed: %v", err)
            }
        }

        // Output JSON to stdout
        enc := json.NewEncoder(os.Stdout)
        enc.SetIndent("", "  ")
        enc.Encode(schema)
    }
    ```

    **Validation (D-16):** When `--validate` flag is passed:
    - For each object type, fetch one page from `beta.peeringdb.com` (D-56)
    - Compare response field names against extracted schema fields
    - Report missing fields (in API but not schema) and extra fields (in schema but not API)
    - Exit with error if critical fields are missing

    **File reading approach:** Read each Python file as text, apply regex patterns per line or per block. No Python runtime dependency.

       **Program structure:**
       ```go
       func main() {
           if len(os.Args) < 2 {
               log.Fatal("usage: pdb-schema-generate <schema.json> [output-dir]")
           }
           schemaPath := os.Args[1]
           outputDir := "ent/schema"
           if len(os.Args) > 2 {
               outputDir = os.Args[2]
           }

           schema, err := loadSchema(schemaPath)
           // ...

           for typeName, typeDef := range schema.ObjectTypes {
               code, err := generateEntSchema(typeName, typeDef)
               // ...
               filename := filepath.Join(outputDir, toSnakeCase(typeDef.ModelName)+".go")
               os.WriteFile(filename, code, 0644)
           }
       }
       ```

       **Code generation logic:**
       For each object type, generate a Go file containing:
       - Package declaration: `package schema`
       - Imports for ent, field, edge, index, entgql

         - `field.Int("id").Positive().Immutable()` per D-37
         - Field mapping: string->field.String, integer->field.Int, float->field.Float, boolean->field.Bool, datetime->field.Time, json_array->field.JSON
         - Optional/Nillable for nullable fields
         - MaxLen for string fields with max_length
         - Default values
         - Unique constraints
         - JSON struct tags per D-36
         - Comments from help_text
       - `Edges()` method with FK references as edges per D-38
       - `Indexes()` method per D-45
       - `Annotations()` method with entgql annotations per D-42
