<objective>
Flip the sync delete pass from hard-delete to soft-delete per D-02. The 13 `deleteStale*` functions in `internal/sync/delete.go` become `markStaleDeleted*` and run `UPDATE ... SET status='deleted', updated=<cycleStart>` instead of `DELETE FROM`. `cycleStart` is captured once at the top of `Worker.Sync` (reusing the existing `start := time.Now()` at worker.go:291) and plumbed through `syncDeletePass` → `syncStep.deleteFn`. The `TestSyncPersistsDeletedRowsUnconditional` test written in Plan 68-01 is replaced by `TestSync_SoftDeleteMarksRows` which asserts the 2-cycle soft-delete round-trip. This is the biggest sync-side change in v1.16.

Purpose: Delivers STATUS-03's prerequisite — rows marked `status=deleted` with a post-deletion `updated` timestamp enable the upstream `rest.py:700-712` `?since>0` query to return tombstones. D-03's one-time gap applies: rows hard-deleted BEFORE this plan ships are gone forever. Tombstone GC policy is deferred to SEED-004.

Output: `internal/sync/delete.go` with 13 renamed functions that run `Update().Where(IDNotIn(...)).SetStatus("deleted").SetUpdated(cycleStart).Save(ctx)`. `internal/sync/worker.go` with updated `syncStep.deleteFn` signature, `syncDeletePass` signature, and `Sync` reusing `start` as `cycleStart`. `internal/sync/integration_test.go` with the 2-cycle test replacing the intermediate one from Plan 68-01.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-68-02-02 | Denial of Service | tombstone table growth | accept → SEED-004 | `deleteStaleChunked`'s fallback at >32K IDs silently no-ops (same as pre-phase). Soft-deleted rows accumulate indefinitely under the current code. SEED-004 is planted for GC policy. |
| T-68-02-03 | Tampering | sync race between primary and replicas | mitigate | LiteFS replication is single-primary; only the primary writes tombstones, replicas receive them via FUSE replication. No new race introduced — pre-phase hard-delete shared the same invariant. |
