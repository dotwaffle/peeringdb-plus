<objective>
Satisfy SYNC-02: prove that running the sync worker with no PeeringDB API key, against a fake upstream
serving the anonymous payload, results in a DB with zero Users-tier rows. Then confirm an anonymous
read through the full production stack returns exactly the row set the worker persisted — the privacy
filter is effectively a no-op because there's nothing to filter.

This is the inverse-direction test of VIS-06/VIS-07: those tests confirm the filter fires correctly
when Users rows ARE in the DB. This test confirms the no-key deployment topology (first-class per
SYNC-02) never produces Users rows in the first place.

Two-step assertion:
1. **Upstream boundary:** fake upstream returns only `visible="Public"` rows (mirrors real upstream
   anon). After sync, `count(Poc where visible='Users') == 0`.
2. **Surface boundary:** hit our local `/api/poc` through the full middleware chain; the response must
   match the set of rows in the DB (filter is a no-op).

Purpose: prevent two regression classes:
- A change that silently serves Users rows in the no-key configuration (e.g. a mis-gated upsert).
- A change that filters out Public rows in the no-key configuration because the filter condition was
  inverted or keyed on the wrong column.

Output: one new test file `internal/sync/nokey_sync_test.go`.
</objective>

    1. Build a fake upstream server that serves the 13 committed anon fixtures when hit at
       `/api/{type}`. Implementation: enumerate `testdata/visibility-baseline/beta/anon/api/` and for
       each type subdirectory, read `page-1.json` bytes at startup. The handler:

       - Matches `GET /api/{type}` (path parsing same as worker_test.go's newFixture).
       - On `skip != 0`, returns an empty-data envelope to terminate pagination (the fixtures are
         single-page for test purposes — this mirrors worker_test.go's simplification).
       - On `skip == 0`, writes the fixture file's bytes directly as the response (already the
         correct JSON envelope). No re-parsing or reshaping.
       - CRITICALLY: on every request, `atomic.AddInt64(&authHeaderCount, 1)` if
         `r.Header.Get("Authorization") != ""`. At end of test, assert this counter is zero — proof
         that the worker never sent an Authorization header, i.e. actually ran anonymous.

       ```go
       var authHeaderCount atomic.Int64
       fixtureDir := "../../testdata/visibility-baseline/beta/anon/api"
       allTypes := []string{"campus", "carrier", "carrierfac", "fac", "ix", "ixfac",
           "ixlan", "ixpfx", "net", "netfac", "netixlan", "org", "poc"}
       fixtures := make(map[string][]byte, len(allTypes))
       for _, typeName := range allTypes {
           b, err := os.ReadFile(filepath.Join(fixtureDir, typeName, "page-1.json"))
           if err != nil {
               t.Fatalf("read fixture %s: %v", typeName, err)
           }
           fixtures[typeName] = b
       }

       srv := httptest.NewServer(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
           if r.Header.Get("Authorization") != "" {
               authHeaderCount.Add(1)
           }
           parts := strings.Split(strings.TrimPrefix(r.URL.Path, "/api/"), "?")
           objType := parts[0]
           skip := r.URL.Query().Get("skip")
           if skip != "" && skip != "0" {
               w.Header().Set("Content-Type", "application/json")
               _, _ = w.Write([]byte(`{"meta":{},"data":[]}`))
               return
           }
           b, ok := fixtures[objType]
           if !ok {
               w.Header().Set("Content-Type", "application/json")
               _, _ = w.Write([]byte(`{"meta":{},"data":[]}`))
               return
           }
           w.Header().Set("Content-Type", "application/json")
           _, _ = w.Write(b)
       }))
       t.Cleanup(srv.Close)
       ```

    4. After the sync, mount `pdbcompat.NewHandler(client).Register(mux)` on a fresh httptest.Server,
       wrapped in the same privctx.TierPublic tier-stamp middleware used in Plan 03. Hit
       `GET /api/poc?limit=50` and assert every returned row matches a DB row (filter is a no-op

       ```go
       mux := http.NewServeMux()
       pdbcompat.NewHandler(client).Register(mux)
       handler := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
           ctx := privctx.WithTier(r.Context(), privctx.TierPublic)
           mux.ServeHTTP(w, r.WithContext(ctx))
       })
       surfaceSrv := httptest.NewServer(handler)
       t.Cleanup(surfaceSrv.Close)

       resp, err := http.Get(surfaceSrv.URL + "/api/poc?limit=50")
       if err != nil || resp.StatusCode != 200 {
           t.Fatalf("GET /api/poc: err=%v status=%d", err, resp.StatusCode)
       }
       body, _ := io.ReadAll(resp.Body)
       resp.Body.Close()

       var env struct {
           Data []map[string]any `json:"data"`
       }
       if err := json.Unmarshal(body, &env); err != nil {
           t.Fatalf("decode /api/poc: %v", err)
       }

       // Every returned row must be in the DB, and every DB row (up to limit) must appear in the
       // response. The filter is a no-op because all DB rows have visible="Public".
       dbPocs, err := client.Poc.Query().Limit(50).All(bypass)
       if err != nil {
           t.Fatalf("query DB POCs: %v", err)
       }
       if len(env.Data) != len(dbPocs) {
           t.Fatalf("surface row count %d != DB row count %d (limit=50)", len(env.Data), len(dbPocs))
       }
       for _, row := range env.Data {
           if vis, _ := row["visible"].(string); vis == "Users" {
               t.Fatalf("visible=Users row surfaced in anonymous response: %+v", row)
           }
       }
       ```

    Constraints & gotchas:
    - The fake upstream must serve ALL 13 types even if the worker only has 13 steps; returning an
      empty-data envelope for missing types IS the terminate-pagination idiom the existing newFixture
      already relies on. No extra effort needed — the fixture map covers all 13.
    - FK integrity: the fixtures are live-captured so child rows (poc, netixlan, etc.) reference
      parent rows (net, ix, ixlan, org) that are ALSO in their own fixtures. The FK-orphan filter in

      whose parent is missing — this is expected behaviour and the reason `totalCount > 0` check
      above is lenient ("at least one POC synced") rather than exact-count. Do NOT assert an exact
      row count — PeeringDB's anon fixtures may have orphaned child rows that get dropped, and the
      exact survivor count is brittle to fixture refresh.
    - The `testdata/visibility-baseline/` directory has mode 0700 on the operator's machine; CI
      checkout restores checkout-default mode. Tests reading with `os.ReadFile` work in both envs.
    - If any fixture file is missing, `t.Fatalf` with the path — don't silently continue. This
      catches checkout / sparse-checkout misconfigurations.

    - Do NOT reuse Plan 03's anon_parity_test.go as a structural diff here. This plan's assertion is

      that. Here the tiny inline privctx.TierPublic stamp is sufficient to exercise the surface.
    - Do NOT spin up multiple sync cycles — one Sync call is enough to assert the invariant.
