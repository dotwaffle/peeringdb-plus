## Map Component Visual Contract

### Placement on facility detail page

The map renders after the detail fields grid and notes, before the collapsible sections (Networks, IXPs, Carriers). Reading order: identity -> location (map) -> relationships. (D-01)

```
Breadcrumb
DetailHeader (fac badge + name + location)
StatBadges (Networks, IXPs, Carriers)
Organization link
Campus link (if applicable)
Detail fields grid (2-column)
Website link
Notes
=== MAP GOES HERE ===       <-- new
CollapsibleSection (Networks)
CollapsibleSection (IXPs)
CollapsibleSection (Carriers)
```

### Map container dimensions

| Breakpoint | Height | Width | Implementation |
|------------|--------|-------|----------------|
| < 768px (mobile) | 200px | 100% content width | `h-[200px]` |
| >= 768px (desktop) | 350px | 100% content width | `md:h-[350px]` |

### Map container styling

```
<div class="w-full h-[200px] md:h-[350px] rounded-lg border border-neutral-200 dark:border-neutral-700 overflow-hidden"
     role="application"
     aria-label="Map showing facility location"
     id="facility-map">
</div>
```

- Full content width (D-03)
- Rounded corners matching existing detail section borders (`rounded-lg`)
- Border: `border border-neutral-200 dark:border-neutral-700` -- consistent with CollapsibleSection border treatment
- `overflow-hidden` to clip map tiles within rounded corners
- No box shadow (discretion: keeps it flat and consistent with other detail sections)

### Conditional rendering

If latitude and longitude are both 0.0, nil, or missing: the map `<div>` does not render at all. No placeholder, no message, no empty container. (D-17, D-18)

Treat `(0.0, 0.0)` as missing data -- no real PeeringDB facility exists at null island. (D-18)

## Tile Layer Contract

### Tile provider

CARTO free public CDN tiles (D-07). No API key required.

| Mode | Style | URL Template |
|------|-------|-------------|
| Light | Voyager | `https://{s}.basemaps.cartocdn.com/rastertiles/voyager/{z}/{x}/{y}{r}.png` |
| Dark | Dark Matter | `https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png` |

Subdomains: `a`, `b`, `c`, `d`

### Attribution

Standard Leaflet attribution control (bottom-right corner). Required text (D-09):

```
&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors &copy; <a href="https://carto.com/attributions">CARTO</a>
```

### Dark mode tile swap

On dark mode toggle (D-08):

1. Listen for clicks on `#dark-mode-toggle` button (same event handler location as existing dark mode toggle in layout.templ)
2. After toggling the `dark` class on `<html>`, check if a Leaflet map instance exists
3. Remove current tile layer, add the opposite theme tile layer
4. Transition: no crossfade -- instant swap (tiles load progressively, which provides its own visual transition)

### Initial tile selection

On page load, read the `dark` class from `<html>` element (set by the existing FOUC-prevention script at top of `<head>`). Select tile URL accordingly.

## Marker & Popup Contract

### Marker

- Standard Leaflet marker with default blue teardrop icon (D-10)
- Marker images loaded from unpkg CDN (Leaflet's default `L.Icon.Default` with `imagePath` set to `https://unpkg.com/leaflet@1.9.4/dist/images/`)
- Single marker per facility detail page (multi-pin is Phase 45)

### Popup content

Popup is closed by default on single-facility maps (D-12). User clicks pin to open.

Popup HTML structure:

```html
<div class="pdb-map-popup">
  <div style="font-weight:600;font-size:14px;margin-bottom:4px;">{facility_name}</div>
  <div style="font-size:12px;color:#666;margin-bottom:4px;">{city}, {country}</div>
  <div style="font-size:12px;color:#666;margin-bottom:6px;">{net_count} Networks &middot; {ix_count} IXPs</div>
  <a href="/ui/fac/{id}" style="font-size:12px;font-weight:500;color:#10b981;">View facility &rarr;</a>
</div>
```

Note: Inline styles used inside Leaflet popups because Tailwind classes may not reliably penetrate Leaflet's popup DOM (Leaflet creates popups outside the main content flow). This is standard practice for Leaflet popup content.

### Popup content fields (D-11)

| Field | Source | Example |
|-------|--------|---------|
| Facility name | `data.Name` | "Equinix LD8" |
| Location | `formatFacLocation(data.City, data.Country)` | "London, GB" |
| Network/IX counts | `data.NetCount`, `data.IXCount` | "42 Networks . 5 IXPs" |
| Detail link | `/ui/fac/{data.ID}` | Always present (D-13) |

### Popup always links to facility detail page (D-13)

This is consistent across all map contexts. On the facility's own page, the link is technically self-referential but maintains template consistency for reuse in Phase 45 (IX/network maps).

## Map Initialization Contract

### Data flow: Go -> JS (D-19)

Latitude and longitude passed from Go templ component to JavaScript via templ's `script` support. The templ component accepts `float64` lat/lng as Go parameters and renders them into a `<script>` block. Type-safe, no extra HTTP fetch.

### MapContainer templ component (D-16)

A shared reusable templ component in a new `map.templ` file (or in `detail_shared.templ`). Accepts Go params and renders the map container `<div>` plus initialization `<script>`.

Component signature (Go):

```go
// MapMarker holds data for a single map pin.
type MapMarker struct {
    Lat       float64
    Lng       float64
    Name      string
    Location  string
    NetCount  int
    IXCount   int
    DetailURL string
}
```

The component accepts a slice of `MapMarker` plus a zoom level. For Phase 44, the slice has exactly one element. Phase 45 will pass multiple markers.

### Leaflet initialization parameters

| Parameter | Value | Source |
|-----------|-------|--------|
| Center | `[lat, lng]` from facility data | D-19 |
| Zoom | 10 | D-04 |
| `scrollWheelZoom` | `false` (enabled on map click) | D-06 |
| `zoomControl` | `true` (default) | D-05 |
| `attributionControl` | `true` (default) | D-09 |

### Scroll wheel zoom behavior (D-06)

```javascript
map.scrollWheelZoom.disable();
map.on('click', function() { map.scrollWheelZoom.enable(); });
map.on('mouseout', function() { map.scrollWheelZoom.disable(); });
```

Prevents accidental zoom while scrolling the page. Enables zoom after intentional click. Disables again when mouse leaves map.

## Accessibility Contract

### Map container ARIA (D-21)

- `role="application"` on map container div
- `aria-label="Map showing facility location"` on map container div

### Keyboard navigation (D-22)

Leaflet's default keyboard support is sufficient:
- `+` / `-` keys for zoom when map is focused
- Arrow keys for panning when map is focused
- No custom keyboard navigation for map panning implemented

### Screen reader considerations

- Map container has descriptive `aria-label`
- Popup content is accessible via Leaflet's built-in popup focus management
- Facility location information is also available in the detail fields above the map (City, Country fields) -- the map is a visual enhancement, not the sole source of location data

## Dark Mode Tile Swap Contract

### Implementation location

The tile swap logic lives in the existing dark mode toggle event handler in layout.templ (the `DOMContentLoaded` script that handles `#dark-mode-toggle` clicks).

### Swap mechanism

```javascript
// After toggling .dark class on <html>:
if (window.__pdbMap && window.__pdbMapTileLayer) {
    var isDark = document.documentElement.classList.contains('dark');
    var newUrl = isDark
        ? 'https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png'
        : 'https://{s}.basemaps.cartocdn.com/rastertiles/voyager/{z}/{x}/{y}{r}.png';
    window.__pdbMapTileLayer.setUrl(newUrl);
}
```

The map initialization script stores the map instance and tile layer on `window` for cross-script access. This is the simplest approach for a non-SPA with inline scripts.

### Global references

| Name | Type | Purpose |
|------|------|---------|
| `window.__pdbMap` | `L.Map` | Leaflet map instance (null if no map on page) |
| `window.__pdbMapTileLayer` | `L.TileLayer` | Current tile layer (for URL swap) |

These are set by the MapContainer init script and consumed by the dark mode toggle handler. Namespaced with `__pdb` prefix to avoid collisions.
