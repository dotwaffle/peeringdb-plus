## Search Results Visual Contract

### Layout change (D-01, D-02, D-03)

Search results change from bordered card layout to compact single-line rows with dividers.

**Before (current):** Each result is an `<a>` with `px-4 py-3 rounded-lg border bg-neutral-800/50` wrapping a two-line flex column (name + subtitle) with a type slug badge on the right.

**After:** Each result is an `<a>` with `px-4 py-2` padding, no border, no background, no border-radius. Results separated by `divide-y divide-neutral-200/50 dark:divide-neutral-700/50` on the parent container. Single-line layout with name left-aligned and metadata right-aligned.

### Result row structure

```
<div class="divide-y divide-neutral-200/50 dark:divide-neutral-700/50">
  <a href="{url}"
     role="option"
     tabindex="-1"
     aria-selected="false"
     class="flex items-center justify-between px-4 py-2 hover:bg-neutral-100 dark:hover:bg-neutral-800/50 transition-colors group focus:outline-none focus:ring-2 focus:ring-emerald-500 focus:ring-offset-1 focus:ring-offset-white dark:focus:ring-offset-neutral-900"
  >
    <span class="text-sm font-medium text-neutral-900 dark:text-neutral-100 truncate min-w-0 group-hover:text-emerald-400 transition-colors">
      {name}
    </span>
    <span class="shrink-0 ml-3 flex items-center gap-3 text-sm text-neutral-500 dark:text-neutral-400 font-mono">
      {metadata badges}
    </span>
  </a>
</div>
```

### Metadata badge ordering (Claude's Discretion)

Right-aligned metadata badges appear in this order, left to right:

1. **Flag + Country code** — `<span class="fi fi-{code}"></span> {CODE}` using existing `CountryFlag` component pattern
2. **City** — plain text, `hidden md:inline` (responsive hide on mobile per D-10)
3. **ASN** — `AS{number}` in `font-mono`, only shown for network results

Each badge separated by `gap-3` (12px). Empty fields omit the badge entirely (no placeholder).

### Hover behavior (Claude's Discretion)

Row hover uses subtle background highlight: `hover:bg-neutral-100 dark:hover:bg-neutral-800/50`. Entity name text shifts to `text-emerald-400` on hover via `group-hover:text-emerald-400`. No card border hover effect (borders removed per D-01).

### Group header (D-03)

Type group header retains the existing accent-colored badge + count display. The per-result type slug badge (`net`, `ix`, `fac`) is removed — the group header already identifies the type.

### Keyboard navigation (D-04)

Keyboard navigation JS preserved with the same `aria-selected="true"` / `ring-2 ring-emerald-500` highlight pattern. Updated selectors to target the new `<a>` elements within `divide-y` containers instead of bordered card elements. Arrow Up/Down moves between results. Enter navigates to selected result. Escape clears selection.

## Comparison Table Visual Contract

### Structure (D-11)

All three comparison sections (IXPs, Facilities, Campuses) convert from `<div>` layouts to `<table>` elements. Tables use the same visual treatment as Phase 43 detail page tables.

```
<div class="overflow-x-auto">
  <table class="w-full text-sm text-left">
    <thead>
      <tr class="bg-neutral-50/50 dark:bg-neutral-800/70 text-xs font-semibold text-neutral-500 dark:text-neutral-400 border-b border-neutral-200 dark:border-neutral-700">
        <th class="px-3 py-2">Column</th>
      </tr>
    </thead>
    <tbody class="divide-y divide-neutral-200/50 dark:divide-neutral-700/50">
      <tr class="hover:bg-neutral-100 dark:hover:bg-neutral-800/50 even:bg-neutral-50/50 dark:even:bg-neutral-800/30">
        <td class="px-3 py-1.5">Value</td>
      </tr>
    </tbody>
  </table>
</div>
```

### Opacity dimming for non-shared rows (D-13)

Non-shared rows in Full View mode: `<tr class="opacity-40">`. Retains existing `compareRowClasses()` function behavior. Applied as a class on the `<tr>` element. Zebra striping and hover still apply beneath the opacity.

### Sorting (D-12)

Comparison tables use the same vanilla JS sort handler from Phase 43 with `data-sortable`, `data-sort-col`, `data-sort-type`, and `data-sort-default` attributes on `<th>` elements. Same sort indicator CSS (emerald triangle `::after` pseudo-element).

### Section container

Each comparison section retains its existing container styling: `bg-neutral-800 rounded-lg border border-neutral-700 overflow-hidden` with a section header bar (`px-4 py-3 border-b border-neutral-700`). The `<table>` replaces the inner `<div class="divide-y">` content.

## Sorting Interaction Contract (Comparison Tables)

Identical to Phase 43 — reuses the existing global sort JS handler from `layout.templ`.

### Click behavior

1. User clicks a sortable `<th>` element
2. First click sorts ascending; second click toggles to descending; third click returns to ascending
3. Only one column is actively sorted at a time

### Sort direction indicator

- CSS `::after` pseudo-element on the active `<th>`
- Ascending: downward-pointing triangle in `text-emerald-400`
- Descending: upward-pointing triangle in `text-emerald-400`
- Inactive sortable columns: faint neutral indicator (`text-neutral-600`)

### Sort data attributes

- Sortable `<td>` elements have `data-sort-value` with raw comparable value
- Numeric columns (Speed, ASN, Shared count): raw integer as string
- Text columns (Name, City, Country): lowercase text
- Empty/missing values: `data-sort-value=""` — sorts last regardless of direction

### Default sort on load

| Table | Default column | Direction |
|-------|---------------|-----------|
| IXP Comparison | IX Name | Ascending |
| Facility Comparison | Name | Ascending |
| Campus Comparison | Campus Name | Ascending |

Tables load pre-sorted server-side. JS applies the active indicator to the default column on load.

## Data Enrichment Contract

### SearchResult struct changes (D-05, D-06)

**Remove:** `Subtitle string` field

**Add:**
- `Country string` — ISO 3166-1 alpha-2 code (e.g. "US", "DE")
- `City string` — city name (e.g. "Frankfurt", "Ashburn")
- `ASN int` — AS number, zero when not applicable (organizations, campuses, carriers)

The parallel struct `SearchHit` in `internal/web/search.go` receives the same field changes.

### Search query enrichment (D-07)

Each entity type's search query adds the necessary selects:
- **Networks:** Country from network fields, City (not typically on network — may need org/facility join), ASN from network ASN field
- **Facilities:** Country and City from facility fields
- **IXPs:** Country and City from IX fields
- **Organizations:** Country and City from org fields
- **Campuses:** Country and City derived from first facility in campus
- **Carriers:** Country from carrier fields
