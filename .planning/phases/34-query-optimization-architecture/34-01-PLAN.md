    **Replace itemToMap with reflect-based field projection (per PERF-05)**

    In `internal/pdbcompat/search.go`:

    1. Add imports: `"reflect"`, `"strings"`, `"sync"` (sync.Once for init).

    2. Define a `fieldAccessor` struct:
       ```go
       type fieldAccessor struct {
           index int
       }
       ```

    3. Define a package-level `fieldMaps` variable:
       ```go
       var fieldMaps sync.Map // map key: reflect.Type, value: map[string]fieldAccessor
       ```

    4. Create a generic `getFieldMap` function that lazily builds the field map for a given type:
       ```go
       func getFieldMap(t reflect.Type) map[string]fieldAccessor {
           if v, ok := fieldMaps.Load(t); ok {
               return v.(map[string]fieldAccessor)
           }
           m := make(map[string]fieldAccessor, t.NumField())
           for i := 0; i < t.NumField(); i++ {
               f := t.Field(i)
               tag := f.Tag.Get("json")
               name, _, _ := strings.Cut(tag, ",")
               if name == "" || name == "-" {
                   continue
               }
               m[name] = fieldAccessor{index: i}
           }
           fieldMaps.Store(t, m)
           return m
       }
       ```

    5. Replace `itemToMap` with a reflect-based version:
       ```go
       func itemToMap(item any) (map[string]any, bool) {
           if m, ok := item.(map[string]any); ok {
               return m, true
           }
           v := reflect.ValueOf(item)
           if v.Kind() == reflect.Ptr {
               if v.IsNil() {
                   return nil, false
               }
               v = v.Elem()
           }
           if v.Kind() != reflect.Struct {
               return nil, false
           }
           fm := getFieldMap(v.Type())
           m := make(map[string]any, len(fm))
           for name, acc := range fm {
               m[name] = v.Field(acc.index).Interface()
           }
           return m, true
       }
       ```

    6. The `applyFieldProjection` function and `isExpandedObject` helper remain unchanged -- they already work with `map[string]any` which `itemToMap` now returns. The _set suffix logic and expanded FK object preservation are unaffected since the reflect version produces the same map keys as the JSON version (both derived from json struct tags).

    7. Remove the `"encoding/json"` import from search.go if it was only used by the old `itemToMap` (check if buildSearchPredicate or other functions need it -- it does not, buildSearchPredicate only uses sql package).
