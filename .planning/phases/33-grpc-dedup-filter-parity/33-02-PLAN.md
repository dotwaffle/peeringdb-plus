**ListParams struct** (declared before ListEntities per CS-6):
```go
// ListParams holds type-specific callbacks for a paginated list query.
// E is the ent entity type, P is the proto message type.
type ListParams[E any, P any] struct {
    EntityName   string
    PageSize     int32
    PageToken    string
    ApplyFilters func() ([]func(*sql.Selector), error)
    Query        func(ctx context.Context, predicates []func(*sql.Selector), limit, offset int) ([]*E, error)
    Convert      func(*E) *P
}
```

**ListEntities function:**
```go
// ListEntities executes a paginated list query using type-specific callbacks.
// Returns converted proto items, next page token, and any error.
func ListEntities[E any, P any](ctx context.Context, params ListParams[E, P]) ([]*P, string, error) {
```
Implementation:
1. Call `normalizePageSize(params.PageSize)` to get pageSize
2. Call `decodePageToken(params.PageToken)` -- on error return `connect.NewError(connect.CodeInvalidArgument, ...)`
3. Call `params.ApplyFilters()` -- return error directly (filter functions return connect errors)
4. Call `params.Query(ctx, predicates, pageSize+1, offset)` -- on error return `connect.NewError(connect.CodeInternal, fmt.Errorf("list %s: %w", params.EntityName, err))`
5. If `len(results) > pageSize`, truncate and create `nextPageToken = encodePageToken(offset + pageSize)`
6. Convert each result via `params.Convert`
7. Return items, nextPageToken, nil

// castPredicates converts generic sql.Selector functions to typed ent predicates.
func castPredicates[T ~func(*sql.Selector)](preds []func(*sql.Selector)) []T {
    out := make([]T, len(preds))
    for i, f := range preds {
        out[i] = T(f)
    }

**Filter function pattern (using sql.FieldEQ/sql.FieldContainsFold per RESEARCH.md recommendation):**

For string fields that are names (name, aka, name_long): use `sql.FieldContainsFold` (case-insensitive substring match, consistent with current behavior)
For all other string fields (status, country, city, website, etc.): use `sql.FieldEQ` (exact match)
For int fields (asn, org_id, etc.): use `sql.FieldEQ` with validation `<= 0` returns InvalidArgument
For bool fields (info_unicast, policy_ratio, etc.): use `sql.FieldEQ`
For id field: use `sql.FieldEQ` with validation `<= 0` returns InvalidArgument

Import the ent field constant packages (e.g., `network`, `carrier`) ONLY for field name constants (e.g., `network.FieldAsn`). Do NOT import the predicate packages anymore since we use sql.Selector functions now.
