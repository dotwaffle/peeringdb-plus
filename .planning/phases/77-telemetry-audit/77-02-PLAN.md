<objective>
Audit current Tempo trace state via Grafana MCP, then introduce a per-route trace sampler that dispatches on URL path prefix to per-group sampling ratios. Implements OBS-07 per CONTEXT.md D-02 (audit + add new sampling rules proactively).

Purpose: Production trace volume is currently dominated by Fly health-check traffic. /healthz and /readyz get hit every ~15s on every machine (8 machines × 2 endpoints × 4/min = 64 traces/min just from health probes), drowning out signal from real traffic. With sampling=1.0, every health probe currently produces a full trace. Per-endpoint sampling lets the operator keep 100% of /api/*, /rest/v1/*, /peeringdb.v1.* traces (the surfaces operators actually debug) while dropping /healthz and /readyz to ~1% (enough for liveness debugging). Cross-service trace continuity is preserved via `sdktrace.ParentBased` — once a parent span samples in, all children sample in regardless of route.

Output: New `internal/otel/sampler.go` with `perRouteSampler`; `internal/otel/provider.go` Setup() wraps it with ParentBased; docs/ARCHITECTURE.md gains a Sampling Matrix subsection; AUDIT.md (created by plan 77-01) gains an OTEL_BSP_* + Tempo-trace-size confirmation appendix.

OTel SDK Sampler interface (`go.opentelemetry.io/otel/sdk/trace`):

```go
// Sampler decides whether a trace should be sampled and exported.
type Sampler interface {
    ShouldSample(parameters SamplingParameters) SamplingResult
    Description() string
}

type SamplingParameters struct {
    ParentContext context.Context
    TraceID       trace.TraceID
    Name          string                  // span name at creation time
    Kind          trace.SpanKind
    Attributes    []attribute.KeyValue    // initial attributes (otelhttp passes RequestTraceAttrs HERE)
    Links         []trace.Link
}

type SamplingResult struct {
    Decision   SamplingDecision   // Drop | RecordOnly | RecordAndSample
    Attributes []attribute.KeyValue
    Tracestate trace.TraceState
}
```

Critical fact for design (verified via context7 + otelhttp v0.68.0 source review):
- otelhttp.NewHandler creates server spans with `trace.WithAttributes(semconv.RequestTraceAttrs(server, r, opts)...)`.
- `RequestTraceAttrs` includes `url.path` (semconv v1.21+) / `http.target` (legacy semconv) at SPAN CREATION TIME.
- Therefore `SamplingParameters.Attributes` contains the URL path BEFORE ShouldSample is called.
- Span NAME at sampling time is the operation string ("peeringdb-plus" in our setup) — NOT the route pattern (Pattern is set later by mux dispatch). Cannot use Name for routing.
