<objective>
Create a project-local override of the entrest `sorting.tmpl` template under `ent/templates/entrest-sorting/` that emits compound `(-updated, -created, -id)` ordering when the request has no explicit `?sort=` parameter (i.e. when the default-sort path is taken). Wire `entc.TemplateDir` in `ent/entc.go` and regenerate `ent/rest/sorting.go`.

Purpose: Per D-02 (updated 2026-04-19) and D-07 (added 2026-04-19), entrest's annotation API cannot express compound default order — `WithDefaultOrder` accepts a single `SortOrder` enum, and the wire protocol is single-field `?sort=<f>&order=<dir>`. Plan 02 gives us the declarative single-field default (`updated DESC`). This plan injects the `created DESC, id DESC` tie-breakers into the generated `applySorting<Type>` bodies so the actual emitted SQL matches pdbcompat and grpcserver.

Output: in-tree template override, wired generator, regenerated `ent/rest/sorting.go` with compound default ORDER BY in all 13 `applySorting<Type>` helpers.
</objective>
