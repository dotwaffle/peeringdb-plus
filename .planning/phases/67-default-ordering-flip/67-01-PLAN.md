<objective>
Add `index.Fields("updated")` to all 13 ent schemas and update the schema-generator template in lock-step so regeneration does not strip the edit. This is the foundation for index-backed `ORDER BY updated DESC, id DESC` queries across pdbcompat, grpcserver, and entrest (addresses D-08 and RESEARCH §G-03).

Purpose: Flipping the default order to `(-updated, -created, -id)` without a supporting index forces a full-table sort on every unbounded list query. Phase 71's `limit=0` budget sizing assumes index-backed ordering.

Output: 13 ent schema files with a new `index.Fields("updated")` entry; `cmd/pdb-schema-generate/main.go` emits the same index in its template; `go generate ./schema` produces no diff.
</objective>
