## Table Visual Contract

### Structure

Every child-entity list renders as a `<table>` inside a `<div class="overflow-x-auto">` wrapper (D-01, D-19).

```
<div class="overflow-x-auto">
  <table class="w-full text-sm text-left">
    <thead>
      <tr class="bg-neutral-50/50 dark:bg-neutral-800/70 text-xs font-semibold text-neutral-500 dark:text-neutral-400 border-b border-neutral-200 dark:border-neutral-700">
        <th class="px-3 py-2">Column</th>
      </tr>
    </thead>
    <tbody class="divide-y divide-neutral-200/50 dark:divide-neutral-700/50">
      <tr class="hover:bg-neutral-100 dark:hover:bg-neutral-800/50 even:bg-neutral-50/50 dark:even:bg-neutral-800/30">
        <td class="px-3 py-1.5">Value</td>
      </tr>
    </tbody>
  </table>
</div>
```

### Table header treatment

- Background: `bg-neutral-50/50 dark:bg-neutral-800/70` — faint, not heavy (D-21)
- Text: `text-xs font-semibold text-neutral-500 dark:text-neutral-400` — smaller than body, semibold
- Bottom border: `border-b border-neutral-200 dark:border-neutral-700`
- NOT sticky — no `position: sticky` (discretion choice: avoids complexity with nested scroll containers inside `<details>` elements; tables within collapsible sections are bounded in length)

### Border treatment

- No vertical cell borders — borderless between columns for clean density (discretion choice)
- Horizontal: `divide-y divide-neutral-200/50 dark:divide-neutral-700/50` between rows
- Outer: table inherits the `border border-neutral-200 dark:border-neutral-700 rounded-lg overflow-hidden` from the parent `CollapsibleSection` container

### Zebra striping

- Even rows: `even:bg-neutral-50/50 dark:even:bg-neutral-800/30` — faint alternating background (D-20)
- Odd rows: transparent (default)

### Row hover

- `hover:bg-neutral-100 dark:hover:bg-neutral-800/50` on `<tr>` elements (D-22)

### Cell padding

- `px-3 py-1.5` on all `<td>` and `<th>` elements (D-23)

## Sorting Interaction Contract

### Click behavior

1. User clicks a sortable `<th>` element
2. First click sorts ascending; second click toggles to descending; third click returns to ascending (D-07, D-09)
3. Only one column is actively sorted at a time — clicking a different column resets direction to ascending

### Sort direction indicator

- CSS `::after` pseudo-element on the active `<th>` (D-08)
- Ascending: downward-pointing triangle (border-top trick) in `text-emerald-400`
- Descending: upward-pointing triangle (border-bottom trick) in `text-emerald-400`
- Inactive sortable columns: faint neutral indicator (`text-neutral-600`) showing both arrows (up/down stacked) to signal interactivity

### Sort data attributes

- Every sortable `<td>` has `data-sort-value` with the raw comparable value (D-12)
- Numeric columns (ASN, Speed): raw integer as string, e.g. `data-sort-value="13335"`
- Text columns (Name, City, Country, Role): lowercase text, e.g. `data-sort-value="cloudflare"`
- Empty/missing values: `data-sort-value=""` — JS sorts these last regardless of direction (D-13)

### Sort cursor

- Sortable `<th>` elements: `cursor-pointer select-none`
- Non-sortable `<th>` elements: `cursor-default`

### Default sort on load

| Table type | Default column | Direction |
|------------|---------------|-----------|
| IX Participants | ASN | Ascending |
| Network IX Presences | IX Name | Ascending |
| Facility lists (all entity types) | Country | Ascending |
| Fac Networks / Org Networks | Name | Ascending |
| Contacts | Role | Ascending |
| IX Prefixes | Prefix | Ascending |

Tables load pre-sorted server-side in the default order. JS applies the active indicator to the default column on load. (D-10)

## Country Flag Contract

### CSS delivery

- flag-icons CSS via CDN `<link>` in layout.templ `<head>` (D-14)
- URL: `https://cdn.jsdelivr.net/gh/lipis/flag-icons/css/flag-icons.min.css`
- Placed after Tailwind browser script, before htmx

### Flag rendering

- Element: `<span class="fi fi-{code}" aria-hidden="true"></span>` where `{code}` is lowercase ISO 3166-1 alpha-2 (D-15)
- Aspect ratio: 4x3 rectangle (default flag-icons behavior) (D-03)
- Size: inline with text, inherits `text-sm` line height. Add `inline-block` if needed for alignment.
- Gap: `gap-1.5` (6px) between flag and country code text
- Accessibility: `aria-hidden="true"` on flag span; country code text provides accessible name

### Missing country

- Empty or missing country code: render empty cell, no flag, no placeholder (D-16)

### Country column cell template

```
<td class="px-3 py-1.5" data-sort-value="{lowercase_code}">
  if country != "" {
    <span class="inline-flex items-center gap-1.5">
      <span class="fi fi-{lowercase_code}" aria-hidden="true"></span>
      <span class="font-mono text-neutral-300">{CODE}</span>
    </span>
  }
</td>
```

| IPv4 | Too wide for mobile — users copy IPs on desktop |
| IPv6 | Too wide for mobile — users copy IPs on desktop |
| RS badge | Minor detail |
| Email | Too wide for mobile |
| Phone | Secondary contact info |

### Always visible on mobile

| Column | Reason |
|--------|--------|
| Name (linked) | Primary identity |
| ASN | Core identifier for networks |
| Country + flag | Geographic context in minimal space |
| Role (contacts) | Primary differentiator |
| Prefix | Primary data for prefix tables |
| Protocol | Needed alongside prefix |

- Responsive hiding: `hidden md:table-cell` on `<th>` and `<td>` (D-18)

- No JS involved — pure CSS (D-18)

## Sort JavaScript Contract

### Placement

Sort JS placed as a `<script>` block at the bottom of layout.templ, after existing keyboard navigation and htmx error handler scripts. Not in individual templ components — single global handler for all tables (discretion choice: consistent with existing JS patterns).

### Behavior specification

```
1. Listen for click events on <th[data-sortable]> elements
2. Read data-sort-col attribute from <th> to identify column index
3. Read data-sort-type attribute: "alpha" or "numeric"
4. Toggle sort direction (asc -> desc -> asc)
5. For each <tr> in <tbody>:
   a. Read data-sort-value from <td> at column index
   b. Empty values sort last regardless of direction
6. Re-append sorted <tr> elements to <tbody> (no DOM recreation)
7. Update ::after indicator on active <th>, clear others
8. On htmx:afterSwap — auto-apply default sort to newly loaded tables
```

### Data attributes on `<th>`

- `data-sortable` — presence indicates this column is sortable
- `data-sort-col="{index}"` — zero-based column index
- `data-sort-type="alpha|numeric"` — comparison strategy
- `data-sort-default="asc|desc"` — if present, this column is the default sort

### CSS for sort indicators

```css
th[data-sortable] { cursor: pointer; user-select: none; position: relative; padding-right: 1.25rem; }
th[data-sortable]::after { content: ''; position: absolute; right: 0.25rem; top: 50%; transform: translateY(-50%); }
th[data-sort-active="asc"]::after { border-left: 4px solid transparent; border-right: 4px solid transparent; border-top: 5px solid #10b981; }
th[data-sort-active="desc"]::after { border-left: 4px solid transparent; border-right: 4px solid transparent; border-bottom: 5px solid #10b981; }
th[data-sortable]:not([data-sort-active])::after { border-left: 3px solid transparent; border-right: 3px solid transparent; border-top: 4px solid #525252; margin-top: 2px; }
```
