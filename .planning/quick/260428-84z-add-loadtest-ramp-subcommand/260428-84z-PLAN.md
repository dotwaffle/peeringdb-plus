<objective>
Add a `ramp` subcommand to `cmd/loadtest` that finds the inflection point — the concurrency at which p95/p99 latency or error rate visibly degrades — for each of the five API surfaces against a peeringdb-plus deployment. Replaces ad-hoc operator probing with a deterministic, scriptable, surface-by-surface capacity probe whose Markdown output can be pasted into incident reports or capacity-planning docs.

Per the user's accepted spec (handoff 2026-04-28): ramp from C=1 by ×1.5/2s; inflection = first step where p95 > 2× baseline OR p99 > 1s OR error rate > 1%; hold 10s past inflection then run 1-2 additional steps; per-surface SEQUENTIAL (not parallel) to eliminate cross-surface contention; default entity=net using /ui/asn/<asn> for the webui surface.

Output: a Markdown table per surface emitted to stdout as soon as that surface's ramp completes — three rows showing baseline (C=1), inflection (C=N), past-inflection (C=2N) — with p50/p95/p99/error-rate/achieved-QPS columns. The ramp tool is a developer/operator tool: NEVER deploy after merge. Tag v1.18.7 only.
</objective>

**Ramp loop sketch** (per surface):
```go
// 1. Run baseline at C=1 for StepDuration; collect samples.
// 2. While step.C < MaxConcurrency:
//      ramp.C = ceil(prev * Growth)  (idempotent guard: ramp.C > prev)
//      run step for StepDuration; collect samples
//      if inflection_detected(step, baseline) {
//        // hold past inflection
//        run step for HoldDuration so we have ≥1000 samples for stable p99
//        // 1-2 steps past inflection
//        for i := 0; i < 2 && step.C < MaxConcurrency; i++ {
//          ramp.C = ceil(prev * Growth); run step for StepDuration
//        }
//        break
//      }
// 3. Emit markdown table for this surface to stdout.
```

**Per-step worker pattern** (no global rate cap — concurrency itself is the variable):
```go
g, gctx := errgroup.WithContext(stepCtx)  // stepCtx is ctx with WithTimeout(stepDuration)
sampleCh := make(chan Result, ramp.C * 16)
for i := 0; i < ramp.C; i++ {
    g.Go(func() error {
        for gctx.Err() == nil {
            ep := pickEndpoint(rampSurface, rampIDs[atomic.AddUint32(&idx,1) % len(rampIDs)])
            sampleCh <- Hit(gctx, cfg.HTTPClient, cfg.Base, cfg.AuthToken, ep)
        }
        return nil
    })
}
go func() { _ = g.Wait(); close(sampleCh) }()
samples := drainSamples(sampleCh)
```

Round-robin through the prefetched ID list (atomic counter mod len(ids)) so requests don't all hit the same row — that would just measure cache-warm path, not the inflection.

**Inflection detector** (purely metric-based on the just-completed step's samples):
```go
func detectInflection(step, baseline stepStats, rcfg RampConfig) (reason string, hit bool) {
    if step.P95 > time.Duration(float64(baseline.P95)*rcfg.P95Multiplier) {
        return fmt.Sprintf("p95 %s > baseline p95 %s × %.1f", step.P95, baseline.P95, rcfg.P95Multiplier), true
    }
    if step.P99 > rcfg.P99Absolute {
        return fmt.Sprintf("p99 %s > %s absolute", step.P99, rcfg.P99Absolute), true
    }
    if step.ErrRate > rcfg.ErrorRateThreshold {
        return fmt.Sprintf("error rate %.2f%% > threshold %.2f%%", step.ErrRate*100, rcfg.ErrorRateThreshold*100), true
    }
    return "", false
}
```

**Test design** (cmd/loadtest/ramp_test.go):
- Stand up a `httptest.NewServer` with a handler that introduces synthetic latency proportional to the in-flight count (atomic counter incremented on entry, decremented on exit, sleep = base + (count * step)). This deterministically reproduces a CPU-bound saturation curve.
- Drive runRamp at the test server's URL with tiny step/hold durations (50ms / 100ms) and small max-concurrency (16) so tests run in under a second.
- Assert: inflection step's C is > 1; baseline step recorded ≥1 sample; markdown contains "baseline" / "inflection" / "past-inflection" labels.
- One test forces inflection via P99 absolute (handler latency exceeds threshold immediately at C>=4).
- One test forces inflection via error rate (handler returns 500 with probability rising in C).
- One test asserts surfaces are run sequentially: instrument the handler to record the time of first request per surface; sort by surface order in --surfaces; assert monotonic timestamps.
