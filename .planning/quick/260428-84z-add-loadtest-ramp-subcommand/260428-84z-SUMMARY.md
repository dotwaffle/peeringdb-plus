# Quick Task 260428-84z: Add loadtest ramp subcommand

## One-liner

Adds a per-surface concurrency-ramp subcommand to `cmd/loadtest` that
discovers the inflection point at which p95/p99 latency or error
rate degrades — replaces ad-hoc operator probing with a
deterministic capacity probe whose markdown output drops straight
into incident reports.

## What ships

**New subcommand**: `loadtest ramp [flags]` joins `endpoints`, `sync`, `soak`.

**Twelve ramp-specific flags** wired in `main.go`:
`--target` (alias for `--base`), `--entity`, `--start`, `--growth`,
`--step-duration`, `--hold-duration`, `--max-concurrency`,
`--p95-multiplier`, `--p99-absolute`, `--error-rate-threshold`,
`--surfaces`, `--prefetch-count`. Defaults match the plan spec
exactly (verified by `TestParseRampFlags_Defaults`).

**Ramp loop** (per surface):

1. Baseline at `--start` for `--step-duration`.
2. Multiply concurrency by `--growth` each step (capped at `--max-concurrency`).
3. **Inflection** triggers on the first step where any of:
   - `p95 > baseline.p95 × --p95-multiplier`
   - `p99 > --p99-absolute`
   - `error rate > --error-rate-threshold`
4. **Hold** at the inflection concurrency for `--hold-duration`.
5. Run 1-2 additional steps past inflection (concurrency permitting).
6. Print markdown table to stdout for the surface.

**Per-surface SEQUENTIAL execution**: surfaces are exercised in
`--surfaces` order, one at a time, so cross-surface contention
does not bias measurements. Verified by
`TestRamp_PerSurface_Sequential` (asserts monotonic first-hit
timestamps + monotonic markdown block ordering).

**Safety**: `rejectUpstreamBase` blocks `--target` pointing at
upstream PeeringDB hosts (`peeringdb.com`, `www.peeringdb.com`,
`auth.peeringdb.com`) before any HTTP fires. The existing safety
banner is still printed at the top of every ramp run.
