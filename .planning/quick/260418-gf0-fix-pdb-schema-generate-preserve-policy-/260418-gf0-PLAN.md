# Quick Task 260418-gf0: Resolve backlog 999.1

## Problem

`cmd/pdb-schema-generate` rewrites `ent/schema/poc.go` from `schema/peeringdb.json` and strips the hand-added `func (Poc) Policy()` method (Phase 59, VIS-04). This forces the `go generate ./ent` workaround instead of `go generate ./...` (documented in CLAUDE.md §Code Generation). Surfaced as Phase 63 executor deviation #1.

## Approach (Option B)

Move `Policy()` + its supporting code into a new non-generated file `ent/schema/poc_policy.go`. The generator touches only `poc.go`; `poc_policy.go` is invisible to it.

Rationale:
- Simpler and more reliable than teaching the generator to preserve hand-edits (marker-based AST merge = tricky).
- Matches the existing Go idiom of splitting schema method sets across files.
- Zero behavior change — `(Poc) Policy()` still registered on the type, same privacy behavior.
