# Phase 45 — UI Design Contract

> Visual and interaction contract for Multi-Pin Maps. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Go/templ/htmx -- not React) |
| Preset | not applicable |
| Component library | none (hand-written templ components) |
| Icon library | Leaflet circleMarkers (colored SVG circles), Leaflet default marker (single-fac only, Phase 44) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Popup content inner gaps, legend dot-to-label gap |
| sm | 8px | Compact element spacing, `gap-2`, legend item vertical gap |
| md | 16px | Map container margin, `px-4`, legend padding |
| lg | 24px | Section padding, `space-y-6` between detail sections |
| xl | 32px | Layout gaps between major page areas |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Multi-pin map container height: 250px mobile, 450px desktop (D-16) -- viewport-relative layout dimensions, not spacing tokens
- Map popup padding: inherits Leaflet default (13px horizontal, 11px top, 13px bottom) -- Leaflet CSS controls popup chrome
- CircleMarker radius: 7px (discretion) -- visual size for multi-pin dots, not a spacing token
- Cluster icon sizes: 30px (small), 40px (medium), 50px (large) -- Leaflet.markercluster convention, not spacing tokens
- Unmapped facility message: 8px margin-top below map container

### Conditional rendering (D-10)

If zero facilities have valid coordinates (all lack lat/lng or all are 0,0): the map `<div>` does not render. No placeholder, no message, no empty container.

### Unmapped facility message (D-11)

When some facilities have coordinates but others do not, render a message below the map container:

```html
<div class="text-xs text-neutral-500 dark:text-neutral-400 mt-2">
    {N} facilities not shown (no location data)
</div>
```

This message renders outside the map container, below it. It is standard templ-rendered HTML (not inside Leaflet DOM), so Tailwind classes apply normally.

## CircleMarker & Popup Contract

### CircleMarker (D-05)

All multi-pin maps use `L.circleMarker` (not the default teardrop marker). This creates a clean colored dot that visually distinguishes "distribution of locations" (multi-pin) from "here is one location" (single-facility Phase 44 teardrop).

```javascript
L.circleMarker([lat, lng], {
    radius: 7,
    fillColor: '{color}',  // from pin color table
    color: '{stroke}',      // from pin color table
    weight: 2,
    opacity: 1,
    fillOpacity: 0.8
}).bindPopup(popupHTML)
```

### Popup content -- IX and network maps (D-14)

```html
<div>
  <div style="font-weight:600;font-size:14px;margin-bottom:4px;">{facility_name}</div>
  <div style="font-size:12px;color:#666;margin-bottom:4px;">{city}, {country}</div>
  <a href="/ui/fac/{id}" style="font-size:12px;color:#10b981;">View facility &rarr;</a>
</div>
```

No network/IX counts in IX/network map popups (D-14). Simpler than Phase 44 facility page popups. Inline styles for Leaflet DOM penetration.

### Popup content -- comparison maps (D-15)

```html
<div>
  <div style="font-weight:600;font-size:14px;margin-bottom:4px;">{facility_name}</div>
  <div style="font-size:12px;color:#666;margin-bottom:4px;">{city}, {country}</div>
```

For shared facilities, the third line shows both network names. For unique facilities, it shows only the present network's name. The `+` separator only appears for shared.

## Bounds & Auto-Fit Contract (D-09)

### fitBounds behavior

All multi-pin maps use `map.fitBounds(bounds, { maxZoom: 13, padding: [20, 20] })` instead of `setView()`.

- `maxZoom: 13` -- city level. Prevents over-zoom when 1-2 nearby pins exist.
- `padding: [20, 20]` -- 20px padding from map edges so edge pins are not clipped.
- Bounds computed from all marker positions using `L.latLngBounds(markerPositions)`.

### Single-pin edge case

If only one facility has valid coordinates, `fitBounds` on a single point sets zoom to `maxZoom` (13). This is acceptable -- it shows the facility at city level.

## Multi-Pin Map Initialization Contract

### Extended MapContainer component

The Phase 44 `MapContainer` templ component and `initMap` script are extended (or a new `initMultiPinMap` script is created) to handle:

1. Multiple markers with color metadata
2. MarkerClusterGroup
3. fitBounds instead of setView
4. CircleMarkers instead of default markers

### Extended MapMarker struct (D-13)

```go
type MapMarker struct {
    Lat       float64
    Lng       float64
    Name      string
    Location  string
    NetCount  int     // retained for Phase 44 compat
    IXCount   int     // retained for Phase 44 compat
    DetailURL string
    Color     string  // circleMarker fillColor hex (e.g. "#10b981")
    Stroke    string  // circleMarker stroke hex (e.g. "#059669")
    Extra     string  // additional popup line (network names for compare)
}
```

### Data flow: Go to JS

The templ component serializes the `[]MapMarker` slice to a JSON array in a `<script>` block. The multi-pin init script reads this array, creates circleMarkers, adds them to a MarkerClusterGroup, fits bounds, and registers on `window.__pdbMaps`.

### Multi-pin init script parameters

| Parameter | Value | Source |
|-----------|-------|--------|
| Element ID | map container div id | templ param |
| Markers JSON | `[{lat, lng, name, location, detailURL, color, stroke, extra}, ...]` | templ JSON serialization |
| Max zoom for fitBounds | 13 | D-09 |
| Show legend | `true` for compare maps, `false` for IX/network maps | templ param |
| Legend labels | `{shared: "Shared", netA: "AS{X} only", netB: "AS{Y} only"}` | templ param (compare only) |

### Scroll wheel zoom behavior

Same as Phase 44:
```javascript
map.scrollWheelZoom.disable();
map.on('click', function() { map.scrollWheelZoom.enable(); });
map.on('mouseout', function() { map.scrollWheelZoom.disable(); });
```

### Dark mode tile swap

Same `window.__pdbMaps` array pattern as Phase 44. The multi-pin init script pushes its tile layer entry to the array. The existing dark mode toggle handler in layout.templ iterates all entries and swaps tile URLs.

## Accessibility Contract

### Map container ARIA

- `role="application"` on map container div
- Context-specific `aria-label` (see table above)

### Keyboard navigation

Leaflet's default keyboard support is sufficient:
- `+` / `-` keys for zoom when map is focused
- Arrow keys for panning when map is focused
- Cluster expand via click only (no keyboard equivalent -- standard Leaflet limitation)

### Screen reader considerations

- Map container has descriptive `aria-label`
- Popup content accessible via Leaflet's built-in popup focus management
- All facility data shown on the map is also available in the table sections below (the map is a visual enhancement, not the sole data source)
- Unmapped facility count message is standard HTML with no ARIA role needed
- Legend is a Leaflet Control (not screen-reader navigable), but pin categories are also visually conveyed by the existing shared/unique row styling in tables below
