Create `internal/web/templates/comparetypes.go` in the `templates` package with these types (following the pattern from detailtypes.go -- all fields exported with doc comments per API-1):

```go
// CompareData holds the full comparison result for template rendering.
type CompareData struct {
    // NetA is the first network's summary info.
    NetA CompareNetwork
    // NetB is the second network's summary info.
    NetB CompareNetwork
    // SharedIXPs lists IXPs where both networks are present.
    SharedIXPs []CompareIXP
    // SharedFacilities lists facilities where both networks are present.
    SharedFacilities []CompareFacility
    // SharedCampuses lists campuses where both networks have facility presence.
    SharedCampuses []CompareCampus
    // AllIXPs lists all IXPs for both networks with a shared flag (for full view).
    AllIXPs []CompareIXP
    // AllFacilities lists all facilities for both networks with shared flag (for full view).
    AllFacilities []CompareFacility
    // ViewMode is "shared" (default) or "full".
    ViewMode string
}

// CompareNetwork holds summary info for one side of the comparison.
type CompareNetwork struct {
    ASN  int
    Name string
    ID   int
}

// CompareIXP holds comparison data for a single IXP showing both networks' presence info.
type CompareIXP struct {
    // IXID is the exchange's PeeringDB ID for cross-linking.
    IXID int
    // IXName is the exchange display name.
    IXName string
    // Shared indicates both networks are present at this IXP.
    Shared bool
    // NetA holds network A's presence details (nil if not present).
    NetA *CompareIXPresence
    // NetB holds network B's presence details (nil if not present).
    NetB *CompareIXPresence
}

// CompareIXPresence holds one network's presence details at an IXP.
type CompareIXPresence struct {
    Speed       int
    IPAddr4     string
    IPAddr6     string
    IsRSPeer    bool
    Operational bool
}

// CompareFacility holds comparison data for a single facility.
type CompareFacility struct {
    // FacID is the facility's PeeringDB ID for cross-linking.
    FacID int
    // FacName is the facility display name.
    FacName string
    // City is the facility's city.
    City string
    // Country is the facility's country code.
    Country string
    // Shared indicates both networks are present at this facility.
    Shared bool
    // NetA holds network A's presence info (nil if not present).
    NetA *CompareFacPresence
    // NetB holds network B's presence info (nil if not present).
    NetB *CompareFacPresence
}

// CompareFacPresence holds one network's presence details at a facility.
type CompareFacPresence struct {
    LocalASN int
}

// CompareCampus holds comparison data for a shared campus.
type CompareCampus struct {
    // CampusID is the campus's PeeringDB ID for cross-linking.
    CampusID int
    // CampusName is the campus display name.
    CampusName string
    // SharedFacilities lists facilities within this campus where both networks are present.
    SharedFacilities []CompareCampusFacility
}

// CompareCampusFacility holds a facility within a shared campus.
type CompareCampusFacility struct {
    FacID   int
    FacName string
}
```

Create `internal/web/compare.go` in the `web` package with:

1. A `CompareService` struct holding `*ent.Client` (following SearchService pattern from search.go).
2. A `NewCompareService(client *ent.Client) *CompareService` constructor.
3. A `CompareInput` struct (per CS-5, since Compare needs ctx + two ASNs + view mode = more than 2 args):
   ```go
   type CompareInput struct {
       ASN1     int
       ASN2     int
       ViewMode string // "shared" or "full"
   }
   ```
4. A `Compare(ctx context.Context, input CompareInput) (*templates.CompareData, error)` method that:

   a. Looks up both networks by ASN using `h.client.Network.Query().Where(network.Asn(input.ASN1)).First(ctx)` (same pattern as handleNetworkDetail in detail.go using First() not Only()). Return a descriptive error wrapping with `%w` if either ASN is not found (per ERR-1). Use `fmt.Errorf("network ASN %d: %w", asn, err)`.

   b. Loads network A's IX presences: `h.client.NetworkIxLan.Query().Where(networkixlan.HasNetworkWith(network.ID(netA.ID))).All(ctx)`.

   c. Loads network B's IX presences with the same pattern.

   d. Loads network A's facility presences: `h.client.NetworkFacility.Query().Where(networkfacility.HasNetworkWith(network.ID(netA.ID))).All(ctx)`.

   e. Loads network B's facility presences with the same pattern.

   f. Computes shared IXPs by building a `map[int]` of IX IDs from network A's presences, then iterating network B's presences to find matches. For each shared IX, create a `CompareIXP` with both networks' presence details.

   g. Computes shared facilities by building a `map[int]` of facility IDs from network A's presences (using `*nf.FacID` with nil guard), then iterating network B's to find matches.

   h. Computes shared campuses: For each network, collect facility IDs. Query facilities that have a campus: `h.client.Facility.Query().Where(facility.IDIn(facIDs...), facility.HasCampus()).WithCampus().All(ctx)`. Build `map[int][]int` (campus ID -> facility IDs) for each network. Intersect campus IDs. For each shared campus, find facility IDs present in both.

   i. If `input.ViewMode == "full"`, also compute `AllIXPs` and `AllFacilities` -- the union of both networks' presences, with the `Shared` flag set on entries that appear in both.

   j. Set `CompareData.ViewMode` to input.ViewMode (defaulting to "shared" if empty).

   k. Return `&templates.CompareData{...}` with all fields populated.

Use `errgroup` (per CC-4) to parallelize the 4 independent presence queries (A's IXPs, B's IXPs, A's facilities, B's facilities). Import `"golang.org/x/sync/errgroup"`. Protect shared state with pre-allocated result slices indexed by goroutine (no mutex needed -- same pattern as search.go's fan-out).

For the "full" view AllIXPs computation: build a map of all IX IDs seen across both networks, iterate to produce the union list. Same for AllFacilities.

Do NOT add VLAN information -- the NetworkIxLan model has no VLAN field (VLAN is on IxLan which is the LAN configuration, not per-network presence). The CONTEXT.md mentions it but the data model does not support it per-network.
