<objective>
Convert the existing live conformance test to anon-vs-anon mode per D-10/D-11: one comparison mode
(upstream anonymous vs our local anonymous), no authenticated path, no API-key secret handling in the
live test. Reuses the existing `-peeringdb-live` gate so CI behaviour is unchanged (the test is
skipped in CI without the flag).

Why remove the authenticated path entirely rather than leave it dormant: D-11 rejects authenticated
conformance in CI (no key, would either skip silently or require fragile secret handling). The
authenticated branch creates two problems if kept: (a) non-anon baseline drift if someone runs with
the key locally, (b) an implicit dependency on a CI-optional env var that's easy to forget about.
Delete it. If authenticated conformance re-emerges as a real requirement, it will land with its own
test file and explicit operator workflow — not as a mode flag on this one.

Purpose: align `internal/conformance/` with the phase 59 read-path floor (our anon responses are the
single shape we commit to matching upstream's anon responses at) and simplify the test's control flow.

Output: updated live_test.go. No new files; no production code changes.
</objective>
