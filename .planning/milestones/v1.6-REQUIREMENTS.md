# Requirements: PeeringDB Plus

**Defined:** 2026-03-24
**Core Value:** Fast, reliable access to PeeringDB data from anywhere in the world, served from the nearest edge node with low latency.

## v1.6 Requirements

Requirements for ConnectRPC / gRPC API milestone. Each maps to roadmap phases.

### Infrastructure

- [x] **INFRA-01**: App listens directly on Fly.io internal port without LiteFS HTTP proxy intermediary
- [x] **INFRA-02**: Sync requests on replicas are replayed to primary via fly-replay response header, gated on Fly.io environment detection
- [x] **INFRA-03**: Sync requests are handled directly (no replay) when not running on Fly.io
- [x] **INFRA-04**: Server supports HTTP/2 cleartext (h2c) alongside HTTP/1.1 via http.Protocols
- [x] **INFRA-05**: fly.toml configured with h2_backend for HTTP/2 to backend

### Proto Generation

- [x] **PROTO-01**: All 13 ent schemas annotated with entproto.Message and entproto.Field for proto generation
- [x] **PROTO-02**: buf toolchain configured (buf.yaml, buf.gen.yaml) with protoc-gen-go + protoc-gen-connect-go
- [x] **PROTO-03**: Proto files generated from ent schemas via entproto with SkipGenFile
- [x] **PROTO-04**: ConnectRPC handler interfaces generated via buf generate

### API Surface

- [x] **API-01**: Get RPC returns a single entity by ID for all 13 PeeringDB types
- [x] **API-02**: List RPC returns paginated results for all 13 PeeringDB types
- [x] **API-03**: List RPCs support typed filter fields for querying
- [x] **API-04**: Service handlers mounted on existing HTTP mux at ConnectRPC path prefix

### Observability & Ecosystem

- [x] **OBS-01**: otelconnect interceptor on all ConnectRPC handlers with WithoutServerPeerAttributes
- [x] **OBS-02**: CORS headers updated for Connect protocol and gRPC-Web content types
- [x] **OBS-03**: gRPC server reflection (v1 and v1alpha) enabled for grpcurl/grpcui discovery
- [x] **OBS-04**: gRPC health check service reports serving status for PeeringDB service

## Future Requirements

### Deferred

- [ ] Expose data via gRPC streaming RPCs — deferred pending demand signal
- [ ] SyncStatus custom RPC — deferred, available via existing REST/GraphQL

## Out of Scope

| Feature | Reason |
|---------|--------|
| protoc-gen-entgrpc service stubs | Hardcoded to google.golang.org/grpc interfaces, incompatible with ConnectRPC |
| Write RPCs (Create/Update/Delete) | Read-only mirror — no write path |
| Bidirectional streaming | No use case for a read-only data mirror |
| Separate gRPC port | ConnectRPC serves all protocols on same port via standard http.Handler |
