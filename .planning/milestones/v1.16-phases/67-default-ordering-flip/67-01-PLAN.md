<objective>
Add three lock-step edits to all 13 ent schemas AND the schema-generator template in one atomic plan:

1. `index.Fields("updated")` in each `Indexes()` method (per D-08) — composite `(updated, id)` via ent's PK-backed ordering — enables index-scan `ORDER BY updated DESC, id DESC`.
2. `entrest.WithDefaultSort("updated")` + `entrest.WithDefaultOrder(entrest.OrderDesc)` in each `Annotations()` method (per D-02, updated 2026-04-19) — declares the entrest single-field default sort.
3. `entrest.WithSortable(true)` on each entity's `updated` field annotation list (per RESEARCH §3) — required or entrest codegen panics at `schema_sorting.go:80`.

All three edits are applied in the same 14 files (13 ent/schema + 1 cmd/pdb-schema-generate/main.go). They are combined into a single plan to avoid the file-write conflict that a separate-plans-in-same-wave arrangement would cause.

Purpose: Establishes the declarative foundation for Phase 67's ordering flip. Index presence is required by Phase 71's memory-budget sizing (assumes index-backed ordering). Default-sort annotations are the REST-wire contract for `?sort=` override behaviour. Without `WithSortable(true)`, Plan 02's template override would fail codegen at the entrest validation step.

Output: 14 files edited, single commit. `go generate ./schema` is idempotent. `go generate ./ent` regenerates `ent/rest/sorting.go` + `ent/migrate/schema.go` to reflect the new defaults and indexes.
</objective>
