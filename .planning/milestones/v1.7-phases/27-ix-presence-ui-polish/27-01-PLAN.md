    **In `detail_shared.templ`:**

    1. Add a `speedColorClass(mbps int)` Go function that returns a Tailwind text color class string based on speed tier (per D-01 speed color tiers from CONTEXT.md):
       - `mbps < 1000` (sub-1G): return `"text-neutral-500"` (gray/muted)
       - `mbps == 1000` (1G): return `"text-neutral-400"` (neutral)
       - `mbps >= 1000 && mbps < 100000` (10G range): return `"text-blue-400"` (blue)
       - `mbps >= 100000 && mbps < 400000` (100G range): return `"text-emerald-400"` (emerald)
       - `mbps >= 400000` (400G+): return `"text-amber-400"` (amber)

    2. Add a `CopyableIP` templ component that renders an IP address with click-to-copy and hover clipboard icon:
       ```
       templ CopyableIP(label string, addr string)
       ```
       Renders: a `<div class="group flex items-center gap-1.5">` containing:
       - `<span class="text-neutral-500 text-xs">` with `label` text ("IPv4" or "IPv6")
       - `<span class="font-mono text-neutral-100 cursor-pointer" onclick="navigator.clipboard.writeText('...')" title="Click to copy">` with `addr`
       - A small clipboard SVG icon `<svg class="w-3.5 h-3.5 text-neutral-600 opacity-0 group-hover:opacity-100 transition-opacity cursor-pointer" ...>` with the same onclick handler
       - The onclick should also briefly show a "Copied!" effect -- use `onclick="navigator.clipboard.writeText('{addr}');this.closest('.group').querySelector('.copied-msg')?.classList.remove('hidden');setTimeout(()=>this.closest('.group').querySelector('.copied-msg')?.classList.add('hidden'),1000)"` and include a `<span class="copied-msg hidden text-xs text-emerald-400 ml-1">Copied!</span>`

    3. Add a `CollapsibleSectionWithBandwidth` templ component with signature:
       ```
       templ CollapsibleSectionWithBandwidth(title string, count int, bandwidth string, loadURL string)
       ```
       This is identical to `CollapsibleSection` but the summary line shows: `title (count) -- bandwidth` where bandwidth is right-aligned. The bandwidth string goes inside the existing `<span class="text-neutral-400 ...">` area as: `<span class="font-mono text-sm text-neutral-400">({count}) &mdash; {bandwidth}</span>` replacing the simple count display. Only show the bandwidth portion if `bandwidth != ""`.

    **In `detailtypes.go`:**

    No struct changes needed -- the existing `NetworkIXLanRow` and `IXParticipantRow` already have `Speed`, `IPAddr4`, `IPAddr6`, `IsRSPeer` fields which are sufficient.

    However, the `NetworkIXLansList` and `IXParticipantsList` templ functions need aggregate bandwidth passed. Since these are fragment components rendered by htmx, the bandwidth info goes in the `CollapsibleSectionWithBandwidth` call in the detail page, not in the list fragment. The handler needs to compute it separately or the detail page needs the count from the main handler. The aggregate bandwidth is already available at the detail page level since we have `IXCount` -- but we do NOT have aggregate speed.

    Add a new field to `NetworkDetail`:
    ```go
    // AggregateBW is the total bandwidth in Mbps across all IX presences (for header display).
    AggregateBW int
    ```

    Add a new field to `IXDetail`:
    ```go
    // AggregateBW is the total bandwidth in Mbps across all participants (for header display).
    AggregateBW int
    ```

    Add a helper Go function in `detail_shared.templ`:
    ```go
    func formatAggregateBW(mbps int) string
    ```
    That formats total bandwidth as short form: e.g., "1.2 Tbps", "480 Gbps", "900 Mbps". Use float division for Tbps: `fmt.Sprintf("%.1f Tbps", float64(mbps)/1_000_000)` when >= 1M, `fmt.Sprintf("%d Gbps", mbps/1000)` when >= 1000, `fmt.Sprintf("%d Mbps", mbps)` otherwise.
