       ```go
       package peeringdb

       // Client fetches data from the PeeringDB API with rate limiting,
       // pagination, and retry logic.
       type Client struct {
           http    *http.Client
           limiter *rate.Limiter
           baseURL string
           logger  *slog.Logger
       }
       ```

       `func NewClient(baseURL string, logger *slog.Logger) *Client`:
       - HTTP client with 30-second timeout per SEC-1
       - Rate limiter: `rate.NewLimiter(rate.Every(3*time.Second), 1)` -- 20 req/min per D-05
       - Store baseURL and logger

       `func (c *Client) FetchAll(ctx context.Context, objectType string) ([]json.RawMessage, error)`:
       - Page size: 250 (maximum verified per D-07)
       - URL format: `{baseURL}/api/{objectType}?limit=250&skip={offset}&depth=0`
       - Loop: skip starts at 0, increments by 250 each iteration
       - Parse response into `Response[json.RawMessage]`
       - Break when `len(resp.Data) == 0`
       - Accumulate all data items into result slice
       - Each request goes through `doWithRetry`
       - Return `(allItems, nil)` on success
       - Wrap errors with context: `fmt.Errorf("fetch %s page %d: %w", objectType, page, err)` per ERR-1
       - Honor ctx.Done() per CTX-2

       `func (c *Client) doWithRetry(ctx context.Context, url string) (*http.Response, error)`:
       - Wait on rate limiter: `c.limiter.Wait(ctx)` per D-05
       - Create request with context: `http.NewRequestWithContext`
       - Set User-Agent: `"peeringdb-plus/1.0"`
       - Max 3 retries per D-10
       - Retry on: HTTP 429 (rate limited), 500, 502, 503, 504
       - Do NOT retry on: 400, 401, 403, 404 (return error immediately)
       - Backoff: 2s, 8s, 32s (exponential with base 2s, multiplier 4x)
       - After max retries: return last error wrapped with context
       - On non-retryable error: return `fmt.Errorf("fetch %s: status %d: %w", url, resp.StatusCode, err)`
       - Check context between retries

       `func (c *Client) FetchType[T any](ctx context.Context, objectType string) ([]T, error)`:
       - Convenience method that calls FetchAll, then unmarshals each json.RawMessage into T
       - Uses `json.Unmarshal` with the target type
       - Per D-08: use `json.Decoder` with `DisallowUnknownFields` = NO (allow unknown fields to be silently dropped)
       - Log unknown field warnings if detectable (this is best-effort; json.Unmarshal silently ignores unknown fields by default, which satisfies D-08)
