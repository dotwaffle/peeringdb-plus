    **Create internal/litefs/primary.go per D-24:**

    Package `litefs`. Imports: `"errors"`, `"os"`, `"strconv"`.

    Define a constant for the standard LiteFS primary file path:
    ```go
    // PrimaryFile is the standard LiteFS lease file path.
    // This file exists ONLY on replica nodes and contains the primary's hostname.
    // Its ABSENCE means this node IS the primary.
    const PrimaryFile = "/litefs/.primary"
    ```

    `IsPrimary() bool`:
    - Calls `IsPrimaryAt(PrimaryFile)`

    `IsPrimaryAt(path string) bool`:
    - `_, err := os.Stat(path)`
    - Returns `errors.Is(err, os.ErrNotExist)` -- file not existing means we ARE primary
    - IMPORTANT: The semantics are inverted from what the filename suggests. The `.primary` file exists on REPLICAS (containing the primary's hostname). Its absence means we are the primary. Document this clearly with a comment per Pitfall 3 in RESEARCH.md.

    `IsPrimaryWithFallback(path string, envKey string) bool`:
    - If `path` file exists (stat succeeds) -> return false (we are replica)
    - If `path` file does NOT exist AND we appear to be running under LiteFS (check if `/litefs` directory exists via os.Stat) -> return true (we are primary)
    - If neither condition (local dev, no LiteFS) -> fall back to `os.Getenv(envKey)` parsed as bool, default true
    - This handles the transition from PDBPLUS_IS_PRIMARY env var (Phase 1) to file-based detection (Phase 3) per Pitfall 6.

    **Tests (internal/litefs/primary_test.go):**

    Use `t.TempDir()` to create temp directories and files for testing. Table-driven per T-1:
    - Create a temp file, call `IsPrimaryAt(tempFilePath)` -> expect false (file exists = replica)
    - Call `IsPrimaryAt(filepath.Join(t.TempDir(), "nonexistent"))` -> expect true (no file = primary)
    - For IsPrimaryWithFallback: test all three branches (file exists, LiteFS dir exists but no file, neither exists with env fallback)

    **Create internal/health/handler.go per D-11, D-13, OPS-04, OPS-05:**

    Package `health`. Imports: `"context"`, `"database/sql"`, `"encoding/json"`, `"net/http"`, `"time"`.

    Define response types:
    ```go
    // Response is the JSON body for health endpoint responses per D-13.
    type Response struct {
        Status     string              `json:"status"`
        Components map[string]Component `json:"components"`
    }

    // Component is the status of an individual health check component.
    type Component struct {
        Status  string `json:"status"`            // "ok", "degraded", "failed"
        Message string `json:"message,omitempty"`
    }
    ```

    `LivenessHandler() http.HandlerFunc`:
    - Returns 200 with `{"status":"ok","components":{}}` always.
    - This confirms the process is running. No checks needed per D-11 (liveness = process alive).

    Define `ReadinessInput` struct per CS-5:
    ```go
    // ReadinessInput holds dependencies for the readiness check handler.
    type ReadinessInput struct {
        DB             *sql.DB
        StaleThreshold time.Duration
    }
    ```

    `ReadinessHandler(in ReadinessInput) http.HandlerFunc`:
    - Create a `Response{Status: "ready", Components: map[string]Component{}}`
    - **DB check**: Call `in.DB.PingContext(ctx)` with a 2-second timeout context. If err, set components["db"] to `{Status: "failed", Message: err.Error()}` and overall status to "not_ready".  If ok, set components["db"] to `{Status: "ok"}`.
    - **Sync check**: Import and call `sync.GetLastSyncStatus(ctx, in.DB)` (the import path is `github.com/dotwaffle/peeringdb-plus/internal/sync`).
      - If err: components["sync"] = `{Status: "failed", Message: err.Error()}`, overall "not_ready"
      - If nil (no sync ever): components["sync"] = `{Status: "failed", Message: "no sync completed"}`, overall "not_ready"
      - If status.Status == "success" and `time.Since(status.LastSyncAt) > in.StaleThreshold`: components["sync"] = `{Status: "degraded", Message: "sync data stale, last sync at <timestamp>, age <duration>"}`, overall "not_ready"
      - If status.Status == "success" and within threshold: components["sync"] = `{Status: "ok", Message: "last sync at <timestamp>, age <duration>"}` where `<timestamp>` is `status.LastSyncAt.UTC().Format(time.RFC3339)` and `<duration>` is `time.Since(status.LastSyncAt).Truncate(time.Second).String()`
      - If status.Status == "running": Check if there's a completed_at (meaning a previous sync exists). Use the same threshold logic on the last completed sync time. If no previous completed sync, treat as "no sync completed".
      - If status.Status == "failed": components["sync"] = `{Status: "degraded", Message: "last sync failed: <error_message>"}`. Still set overall to "not_ready" if the LastSyncAt is beyond threshold.
    - Write HTTP status: 200 if `resp.Status == "ready"`, 503 if `"not_ready"`.
    - Set `Content-Type: application/json`. Encode response with `json.NewEncoder(w).Encode(resp)`.

    NOTE: The sync check requires importing the sync package. To avoid circular dependencies, the health package imports sync (sync does NOT import health). This is fine -- sync is a lower-level package.

Call `sync.InitStatusTable` to create the table. Insert test rows directly with `db.ExecContext` to simulate various sync states.

    For the "last sync timestamp exposed" (OPS-05), verify that the sync component's Message contains a timestamp string matching RFC3339 format.
