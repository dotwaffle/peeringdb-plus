    - HEAD request with valid sync time: same Cache-Control and ETag headers as GET
    - POST request: no Cache-Control or ETag headers, inner handler called normally
    - GET request with zero sync time (no sync yet): no Cache-Control or ETag headers, inner handler called normally
    - GET with `If-None-Match` matching current ETag: returns 304 Not Modified with ETag header, no body, inner handler NOT called
    - GET with `If-None-Match: *`: returns 304 Not Modified
    - GET with `If-None-Match` not matching: normal response with Cache-Control and ETag headers
    - ETag changes when sync timestamp changes
    - ETag is properly quoted as weak ETag: `W/"hexdigest"`
  </behavior>

       - `SyncInterval time.Duration` -- configured sync interval for max-age calculation

    2. Implement `Caching(in CachingInput) func(http.Handler) http.Handler`:
       - Only applies to GET and HEAD methods; all others pass through unchanged
       - Calls `in.SyncTimeFn()` to get current sync time
       - If zero time: skip caching headers, call next handler
       - Compute ETag via `computeETag(syncTime)`: SHA-256 hash of `syncTime.Format(time.RFC3339Nano)`, truncated to 16 bytes, formatted as `W/"<32-hex-chars>"`
       - Set `Cache-Control: public, max-age=N` where N = `int(in.SyncInterval.Seconds()) + 120`
       - Set `ETag: W/"<hash>"`
       - Check `If-None-Match` header: if matches current ETag (or is `*`), write 304 and return immediately (no body, no next handler call)
       - Otherwise call next handler (headers already set before delegation)

    3. Implement unexported helpers:

       - `etagMatch(ifNoneMatch, etag string) bool` -- trimmed comparison, handles `*` wildcard
