    **Per SRVR-04 and CONTEXT.md decisions: 1 MB limit via http.MaxBytesReader on POST endpoints.**

    1. Add a constant near the top of main.go (after the import block, before `func init()`):
       ```go
       // maxRequestBodySize is the maximum allowed request body for POST endpoints (1 MB).
       // GraphQL queries rarely exceed 10 KB; 1 MB is generous per SRVR-04.
       const maxRequestBodySize = 1 << 20
       ```

    2. Modify the POST /sync handler registration (line 172). Wrap the sync handler with body limiting. Replace:
       ```go
       mux.HandleFunc("POST /sync", newSyncHandler(ctx, SyncHandlerInput{...}))
       ```
       with:
       ```go
       syncHandler := newSyncHandler(ctx, SyncHandlerInput{
           IsPrimaryFn: isPrimaryFn,
           SyncToken:   cfg.SyncToken,
           DefaultMode: cfg.SyncMode,
           SyncFn: func(syncCtx context.Context, mode config.SyncMode) {
               syncWorker.SyncWithRetry(syncCtx, mode)
           },
       })
       mux.HandleFunc("POST /sync", func(w http.ResponseWriter, r *http.Request) {
           r.Body = http.MaxBytesReader(w, r.Body, maxRequestBodySize)
           syncHandler(w, r)
       })
       ```

    3. Modify the /graphql handler (lines 193-199). Add body limiting for POST only:
       ```go
       playgroundHandler := pdbgql.PlaygroundHandler("/graphql")
       mux.HandleFunc("/graphql", func(w http.ResponseWriter, r *http.Request) {
           if r.Method == http.MethodGet {
               playgroundHandler.ServeHTTP(w, r)
               return
           }
           r.Body = http.MaxBytesReader(w, r.Body, maxRequestBodySize)
           gqlHandler.ServeHTTP(w, r)
       })
       ```

    Note: http.MaxBytesReader will cause the handler to receive an error when reading past the limit. The net/http server automatically returns 413 when a MaxBytesError is detected during request body reads in Go 1.22+. Log at WARN level if the body-too-large error surfaces (Claude's discretion per CONTEXT.md). The existing error handling in gqlHandler and syncHandler will surface the error naturally.
