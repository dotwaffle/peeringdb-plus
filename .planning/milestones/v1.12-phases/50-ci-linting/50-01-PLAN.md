Add a new `docker-build` job to .github/workflows/ci.yml that runs in parallel with the existing lint, test, build, and govulncheck jobs (per D-QUAL-04 from CONTEXT.md).

The new job should be added after the `govulncheck` job block. Structure:

```yaml
  docker-build:
    name: Docker Build
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v6

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build development image
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ./Dockerfile
          push: false
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Build production image
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ./Dockerfile.prod
          push: false
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

Key details:
- Uses docker/setup-buildx-action@v3 for buildx support and layer caching
- Uses docker/build-push-action@v6 with `push: false` (build-only, no registry push)
- GHA cache (type=gha) for layer caching between CI runs
- Builds both Dockerfile (dev, CGO_ENABLED=0) and Dockerfile.prod (with LiteFS)
- No Go setup needed -- Docker handles the Go toolchain internally
- Job has no `depends_on`/`needs` -- runs in parallel with all other jobs
- Follows existing conventions: actions/checkout@v6, ubuntu-latest runner
