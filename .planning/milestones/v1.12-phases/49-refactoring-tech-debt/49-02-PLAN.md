Refactor upsert.go per REFAC-02 from CONTEXT.md decisions. The approach from the specifics section:

1. Create a generic `upsertBatch` function that encapsulates the batch loop:
```go
// upsertBatch splits builders into batches of batchSize and executes saveFn
// for each batch. Returns collected IDs from the idsFn applied to each input item.
func upsertBatch[Item any, Builder any](
    ctx context.Context,
    items []Item,
    idFn func(Item) int,
    buildFn func(Item) Builder,
    saveFn func(context.Context, []Builder) error,
    entityName string,
) ([]int, error) {
    ids := make([]int, 0, len(items))
    builders := make([]Builder, 0, len(items))
    for _, item := range items {
        ids = append(ids, idFn(item))
        builders = append(builders, buildFn(item))
    }
    for i := 0; i < len(builders); i += batchSize {
        end := min(i+batchSize, len(builders))
        if err := saveFn(ctx, builders[i:end]); err != nil {
            return nil, fmt.Errorf("upsert %s batch %d: %w", entityName, i/batchSize, err)
        }
    }
    return ids, nil
}
```

2. Refactor each of the 13 upsert functions to use `upsertBatch`. Each function keeps its existing signature `func upsertXxx(ctx context.Context, tx *ent.Tx, items []peeringdb.Xxx) ([]int, error)` so worker.go call sites do NOT need to change. Internally, each function calls `upsertBatch` with:
   - `idFn`: extracts `.ID` from the PeeringDB struct
   - `buildFn`: creates the ent builder with all field setters (this is the per-type logic that differs)
   - `saveFn`: calls `tx.Entity.CreateBulk(batch...).OnConflictColumns(entity.FieldID).UpdateNewValues().Exec(ctx)`
   - `entityName`: string for error messages (e.g., "organizations", "networks")

Example refactored function:

```go
            return tx.Organization.CreateBulk(batch...).
                OnConflictColumns(organization.FieldID).
                UpdateNewValues().
                Exec(ctx)
        },
        "organizations",
    )
}
```
