**Step 2: Create `internal/pdbcompat/golden_test.go` with the following structure:**

**Package-level update flag (per CONTEXT.md decision):**
```go
var update = flag.Bool("update", false, "update golden files")
```

**`compareOrUpdate` helper function:**
- Takes `t *testing.T`, `goldenPath string`, `got []byte`
- If `*update` is true: `os.MkdirAll(filepath.Dir(goldenPath), 0o755)` then `os.WriteFile(goldenPath, got, 0o644)` then return
- Otherwise: `os.ReadFile(goldenPath)` and compare using `cmp.Diff(string(want), string(got))`
- On read error: `t.Fatalf("read golden file %s: %v (run with -update to create)", goldenPath, err)`
- On diff: `t.Errorf("golden mismatch for %s (-want +got):\n%s", goldenPath, diff)`

**`setupGoldenTestData` function (per CONTEXT.md: separate from setupTestHandler, uses fixed timestamps and explicit IDs):**
- Signature: `func setupGoldenTestData(t *testing.T) (*http.ServeMux, map[string]int)`
- Returns the mux and a map of type name to entity ID for detail/depth requests
- Fixed timestamp: `var goldenTime = time.Date(2025, 1, 1, 0, 0, 0, 0, time.UTC)`
- Uses `testutil.SetupClient(t)` for in-memory SQLite
- Creates ALL 13 entity types with explicit IDs (use `SetID()`) and fixed timestamps
- Entity creation order (respecting FK constraints):
  1. Organization (ID=100): Name="Golden Org", Aka="GO", Status="ok"
  2. Campus (ID=200): Name="Golden Campus", linked to org 100
  3. Facility (ID=300): Name="Golden Facility", linked to org 100 and campus 200
  4. Network (ID=400): Name="Golden Net", ASN=65000, linked to org 100
  5. InternetExchange (ID=500): Name="Golden IX", linked to org 100
  6. IxLan (ID=600): linked to IX 500
  7. Carrier (ID=700): Name="Golden Carrier", linked to org 100
  8. Poc (ID=800): Name="Golden Contact", linked to net 400
  9. IxPrefix (ID=900): Prefix="192.0.2.0/24", Protocol="IPv4", linked to ixlan 600
  10. NetworkFacility (ID=1000): linked to net 400 and fac 300
  11. NetworkIxLan (ID=1100): linked to net 400, ixlan 600, IX 500
  12. IxFacility (ID=1200): linked to IX 500 and fac 300
  13. CarrierFacility (ID=1300): linked to carrier 700 and fac 300
- ALL entities use `SetCreated(goldenTime).SetUpdated(goldenTime).SetStatus("ok")`
- Use simple, fixed values for all string fields (avoid floats with many decimals per Pitfall 3)
- For latitude/longitude use clean values like `37.5` and `-122.5`
- Return mux with handler registered, and ids map like: `{"org": 100, "net": 400, "fac": 300, ...}`

**`TestGoldenFiles` function (per CONTEXT.md: table-driven across all 13 types x 3 scenarios):**
- Uses `t.Parallel()` at top level
- Calls `setupGoldenTestData(t)` ONCE (shared across subtests since data is read-only)
- Define scenarios as `[]struct{ name, urlFmt string }` where:
  - `list`: URL = `/api/{type}`
  - `detail`: URL = `/api/{type}/{id}`
  - `depth`: URL = `/api/{type}/{id}?depth=2`
- Iterate all 13 types from Registry (sorted for deterministic order) x 3 scenarios
- For each subtest `t.Run(typeName+"/"+scenario.name, ...)`:
  - Use `t.Parallel()` on each subtest
  - Build URL using the ids map for detail/depth (list does not need ID)
  - Create `httptest.NewRequest` and `httptest.NewRecorder`
  - Call `mux.ServeHTTP(rec, req)`
  - Assert `rec.Code == http.StatusOK`
  - Build golden path: `filepath.Join("testdata", "golden", typeName, scenario.name+".json")`
  - Call `compareOrUpdate(t, goldenPath, rec.Body.Bytes())`

**Important implementation notes:**
- Use `sort.Strings()` on Registry keys before iterating for deterministic test order
- The `rec.Body.Bytes()` captures the exact output including trailing `\n` from `json.NewEncoder.Encode()` (Pitfall 2)
- For `detail` and `depth` URLs, use the ids map to get the entity ID for each type
- All golden file paths use the format `{type}/{scenario}.json` per CONTEXT.md decision
- JSON format is compact (not pretty-printed) per CONTEXT.md decision
