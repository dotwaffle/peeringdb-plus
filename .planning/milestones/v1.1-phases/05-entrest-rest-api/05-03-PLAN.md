<interfaces>
<!-- entrest filter annotation API (from entrest schema_filtering.go source) -->

Individual filter predicates (bitwise OR to combine):
  entrest.FilterEQ          -- Equality (field.eq=value)
  entrest.FilterNEQ         -- Not equal (field.neq=value)
  entrest.FilterGT          -- Greater than (field.gt=value)
  entrest.FilterGTE         -- Greater than or equal (field.gte=value)
  entrest.FilterLT          -- Less than (field.lt=value)
  entrest.FilterLTE         -- Less than or equal (field.lte=value)
  entrest.FilterIsNil       -- IS NULL (field.is_nil=true)
  entrest.FilterIn          -- In list (field.in=a,b,c)
  entrest.FilterNotIn       -- Not in list (field.not_in=a,b,c)
  entrest.FilterEqualFold   -- Case-insensitive equality
  entrest.FilterContains    -- Contains substring (field.has=value)
  entrest.FilterContainsFold -- Case-insensitive contains
  entrest.FilterHasPrefix   -- Starts with (field.has_prefix=value)
  entrest.FilterHasSuffix   -- Ends with (field.has_suffix=value)

Pre-built filter groups (combine common predicates):
  entrest.FilterGroupEqualExact -- EQ | NEQ | EqualFold | IsNil
  entrest.FilterGroupEqual      -- EqualExact | Contains | HasPrefix | HasSuffix
  entrest.FilterGroupContains   -- Contains | ContainsFold | IsNil
  entrest.FilterGroupLength     -- GT | LT
  entrest.FilterGroupArray      -- In | NotIn

Edge filter:
  entrest.FilterEdge -- Applied to edges for has/hasNot filtering

Usage on fields:
  field.String("name").Annotations(entrest.WithFilter(entrest.FilterGroupEqual | entrest.FilterGroupArray))
  field.Int("asn").Annotations(entrest.WithFilter(entrest.FilterEQ | entrest.FilterGT | entrest.FilterLT | entrest.FilterIn))

Generated result: ListXxxParams will embed Filtered[predicate.Xxx] and have typed filter fields
like NameEQ *string, NameHas *string, AsnEQ *int, AsnGT *int, etc.

Query parameter format: field.predicate=value (e.g., ?name.eq=Cloudflare, ?asn.gt=1000, ?name.has=cloud)
</interfaces>

    Add entrest.WithFilter() annotations to key fields across all 13 schemas. The original D-05 decision assumed entrest would auto-discover filterable fields, but it requires explicit per-field WithFilter annotations. This is a correction to that assumption, not a violation of the decision's intent.

    **Filter annotation strategy by field type:**

    For String fields that users commonly search (name, aka, country, city, status, role):
    - Use `entrest.WithFilter(entrest.FilterGroupEqual | entrest.FilterGroupArray)` -- provides eq, neq, contains, has_prefix, has_suffix, in, not_in, equal_fold, is_nil

    For Int fields (asn, FK IDs like org_id, net_id, speed, mtu, counts):
    - Use `entrest.WithFilter(entrest.FilterEQ | entrest.FilterNEQ | entrest.FilterGT | entrest.FilterGTE | entrest.FilterLT | entrest.FilterLTE | entrest.FilterIn | entrest.FilterNotIn)` -- provides numeric equality and range queries

    For Bool fields (info_unicast, info_ipv6, is_rs_peer, operational, etc.):
    - Use `entrest.WithFilter(entrest.FilterEQ)` -- provides boolean equality (true/false)

    For Time fields (created, updated):
    - Use `entrest.WithFilter(entrest.FilterGT | entrest.FilterGTE | entrest.FilterLT | entrest.FilterLTE)` -- provides time range queries

    Do NOT add WithFilter to:
    - JSON fields (social_media, info_types, available_voltage_services) -- JSON fields cannot be filtered by entrest
    - Long text fields (notes, website URLs) -- not useful for filtering
    - Fields already carrying multiple annotations where stacking WithFilter alongside WithSchema may conflict

    **Per-schema field list (add WithFilter to these fields):**

    Organization: name, aka, name_long, city, state, country, status
    Network: name, aka, name_long, asn, org_id, info_type, info_traffic, info_ratio, info_scope, info_unicast, info_ipv6, policy_general, status, country (via computed fields if present), created, updated
    Facility: name, aka, name_long, org_id, campus_id, city, state, country, status, created, updated
    InternetExchange: name, aka, name_long, org_id, city, country, region_continent, media, proto_unicast, proto_ipv6, status, created, updated
    IxLan: name, ix_id, mtu, status, created, updated
    IxPrefix: ixlan_id, protocol, prefix, in_dfz, status, created, updated
    IxFacility: ix_id, fac_id, name, city, country, status, created, updated
    NetworkFacility: net_id, fac_id, name, city, country, local_asn, status, created, updated
    NetworkIxLan: net_id, ixlan_id, ix_id, name, asn, speed, ipaddr4, ipaddr6, is_rs_peer, operational, status, created, updated
    Carrier: name, aka, name_long, org_id, status, created, updated
    CarrierFacility: carrier_id, fac_id, name, status, created, updated
    Campus: name, aka, org_id, city, state, country, status, created, updated
    Poc: net_id, role, visible, name, email, status, created, updated

    When a field already has Annotations() (e.g., name fields with entgql.OrderField or social_media with entrest.WithSchema), add the WithFilter annotation alongside using comma-separated annotations. Example:
    ```go
    field.String("name").
        MaxLen(255).
        NotEmpty().
        Unique().
        Annotations(
            entgql.OrderField("NAME"),
            entrest.WithFilter(entrest.FilterGroupEqual | entrest.FilterGroupArray),
        ).
        Comment("Network name"),
    ```
