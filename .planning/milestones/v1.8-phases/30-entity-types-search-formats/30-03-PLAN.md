
The renderer receives `[]templates.SearchGroup` directly (already computed by SearchService).

Structure per D-05, D-06, D-07 and research pattern:
1. Title line: `Search: "{query}"` -- but we don't have the query string in the data. Use a simpler heading: just iterate groups. The search query is not passed to the renderer. Use `StyleHeading.Render("Search Results")` as the main heading.
   Actually, looking at the data flow: `handleHome` sets `Data: groups` and `Title: "Search"`. The `RenderPage` method receives `title string` but `RenderSearch` only gets `groups`. To include the query, we'd need to modify the signature. Instead, just render the groups without echoing the query -- the user already knows what they searched for. Start with a blank line before the first group.

2. If `len(groups) == 0`, show "No results found." in StyleMuted.

3. For each group:
   - Header: `StyleHeading.Render(fmt.Sprintf("%s (%d results)", group.TypeName, group.TotalCount))`
   - For each result in group.Results:
     - `  StyleValue.Render(result.Name)` + if result.Subtitle != "" { `  StyleMuted.Render(result.Subtitle)` } + `  StyleLink.Render(result.DetailURL)`
   - Blank line between groups

4. Final newline.
