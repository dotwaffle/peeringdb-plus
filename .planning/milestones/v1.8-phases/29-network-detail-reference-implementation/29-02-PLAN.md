**Replace the RenderNetworkDetail stub** in network.go with the full implementation. The method renders a whois-style network detail page for terminal clients.

**Layout structure per locked decisions:**

1. **Title line (bold heading):** `{Name}  AS{ASN}` -- Name in StyleHeading, ASN in StyleMuted (D-01)

2. **Key-value header** using writeKV with right-aligned labels (D-01, D-02):
   - Compute `labelWidth` as the maximum label length among displayed fields. The labels are: "Type", "Peering Policy", "Website", "Organization", "IRR AS-SET", "Looking Glass", "Route Server", "Traffic", "Ratio", "Scope", "IX Presences", "Facilities", "Prefixes v4", "Prefixes v6", "Aggregate Bandwidth". The longest is "Aggregate Bandwidth" at 19 chars. Use `const labelWidth = 19`.
   - Field ordering (Claude's discretion): Type, Peering Policy, Website, Organization, IRR AS-SET, Looking Glass, Route Server, Traffic, Ratio, Scope, IX Presences, Facilities, Prefixes v4, Prefixes v6, Aggregate Bandwidth
   - For Peering Policy: use `PolicyStyle(data.PolicyGeneral)` as the value (already styled, do not wrap in StyleValue) (D-03, RND-13)
   - For IX Presences/Facilities: use `strconv.Itoa(data.IXCount)` and `strconv.Itoa(data.FacCount)` as values
   - For Prefixes: use `strconv.Itoa(data.InfoPrefixes4)` and `strconv.Itoa(data.InfoPrefixes6)`. Omit if zero.
   - For Aggregate Bandwidth: use `FormatBandwidth(data.AggregateBW)`. **Omit if AggregateBW == 0** (Pitfall 6 from research). (RND-15)
   - For all other fields: pass through `StyleValue.Render(value)` before writeKV
   - Note: writeKV already omits empty string values. For int fields that should be omitted when zero, convert to string and only pass if > 0.

3. **IX Presences section** (D-04, D-05, D-07, RND-14, RND-16):
   - Only render if `len(data.IXPresences) > 0`
   - Section header: `"\nIX Presences ({count})"` in StyleHeading, followed by `"  {FormatBandwidth(sectionBW)}"` in StyleMuted if section BW > 0. Compute section BW by summing `row.Speed` across all IXPresences.
   - Newline after header.
   - Each IX on one line (D-04), indented with two spaces:
     - `"  {IXName} [/ui/ix/{IXID}]"` -- IXName in StyleValue, path via CrossRef (D-08, D-09, RND-16)
     - If `row.IsRSPeer`: append `"  [RS]"` styled with `lipgloss.NewStyle().Foreground(ColorSuccess)` (D-07, RND-14)
     - If `row.Speed > 0`: append `"  {FormatSpeed(row.Speed)}"` styled via `SpeedStyle(row.Speed).Render(FormatSpeed(row.Speed))` (D-05, RND-12)
     - If `row.IPAddr4 != ""`: append `"  {IPAddr4}"`
     - If both IPs: append `" / "` between them; if only IPv6: append `"  {IPAddr6}"`
     - Newline at end of each row

4. **Facilities section** (D-04, D-06, RND-16):
   - Only render if `len(data.FacPresences) > 0`
   - Section header: `"\nFacilities ({count})"` in StyleHeading
   - Newline after header.
   - Each facility on one line, indented with two spaces:
     - `"  {FacName} [/ui/fac/{FacID}]"` -- FacName in StyleValue, path via CrossRef (D-08, D-09, RND-16)
     - If FacID == 0: omit the CrossRef (means FacID was nil in DB, per Pitfall 4)
     - If City or Country non-empty: append `"  {City}, {Country}"` in StyleMuted
     - Newline at end of each row

5. **Final newline** for clean terminal output.

6. **Call `r.Write(w, buf.String())`** to apply colorprofile stripping for Plain/noColor modes.

**Performance note (from research Pitfall 2):** For networks with 1000+ IX presences, pre-allocate the builder:

```go
buf.Grow(len(data.IXPresences)*120 + len(data.FacPresences)*80 + 500)
```
