3. Create `buf.gen.yaml` at project root with this exact content:
   ```yaml
   version: v2
   clean: true
   managed:
     enabled: true
     override:
       - file_option: go_package_prefix
         value: github.com/dotwaffle/peeringdb-plus/gen
   plugins:
     - local: [go, tool, protoc-gen-go]
       out: gen
       opt:
         - paths=source_relative
     - local: [go, tool, protoc-gen-connect-go]
       out: gen
       opt:
         - paths=source_relative
         - simple
   ```

4. Create directory `proto/peeringdb/v1/` and write `common.proto` with this exact content:
   ```protobuf
   syntax = "proto3";
   package peeringdb.v1;
   option go_package = "github.com/dotwaffle/peeringdb-plus/gen/peeringdb/v1";

   // SocialMedia represents a social media link from PeeringDB.
   // Used by Organization, Network, Facility, InternetExchange, Carrier, and Campus.
   message SocialMedia {
     string service = 1;
     string identifier = 2;
   }
   ```

The `simple` option on protoc-gen-connect-go produces cleaner handler signatures: `(ctx, *Request) -> (*Response, error)` instead of `(ctx, *connect.Request[Req]) -> (*connect.Response[Resp], error)`. This matches Go idioms and simplifies Phase 23 service implementations.

<task type="auto">
  <name>Task 2: Wire entproto extension into entc.go</name>
  <files>ent/entc.go</files>
  <read_first>ent/entc.go, go.mod</read_first>
  <action>
Modify `ent/entc.go` to add the entproto extension alongside the existing entgql and entrest extensions.

1. Add import: `"entgo.io/contrib/entproto"`

2. After the `restExt` creation block (after line ~76), add a new entproto extension block:
   ```go
   protoExt, err := entproto.NewExtension(
       entproto.SkipGenFile(),
       entproto.WithProtoDir("../proto/peeringdb/v1"),
   )
   if err != nil {
       log.Fatalf("creating entproto extension: %v", err)
   }
   ```

   - `SkipGenFile()` prevents entproto from generating `//go:generate protoc ...` directives -- we use buf instead.
   - `WithProtoDir("../proto/peeringdb/v1")` sets the output path relative to `ent/` where entc.go runs. The `../` is needed because generation runs from the `ent/` directory, so `../proto/peeringdb/v1` resolves to `proto/peeringdb/v1/` at project root.

3. Update the `opts` slice to include protoExt:
   ```go
   opts := []entc.Option{
       entc.Extensions(gqlExt, restExt, protoExt),
       entc.FeatureNames("sql/upsert"),
   }
   ```
   Change from `entc.Extensions(gqlExt, restExt)` to `entc.Extensions(gqlExt, restExt, protoExt)`.

Do NOT add `entproto.Service()` -- that generates standard gRPC service stubs, which we do not want. We write ConnectRPC services manually in Phase 23.
  </action>
