For each schema, make three types of changes:

**A. Add `entproto.Message(entproto.PackageName("peeringdb.v1"))` to the Annotations() method.**

Add it to the existing annotation slice. Example:
```go
func (Organization) Annotations() []schema.Annotation {
    return []schema.Annotation{
        entgql.RelayConnection(),
        entgql.QueryField(),
        entrest.WithIncludeOperations(entrest.OperationRead, entrest.OperationList),
        entproto.Message(entproto.PackageName("peeringdb.v1")),
    }
}
```

**B. Add `entproto.Field(N)` to every non-ID field, and `entproto.Skip()` to social_media fields.**

Field number 1 is auto-assigned to `id` by entproto -- do NOT annotate the `id` field. Start numbering at 2 for the first non-ID field, incrementing sequentially in declaration order.

For fields that already have `Annotations(...)`, add `entproto.Field(N)` or `entproto.Skip()` to the existing Annotations call. For fields without existing Annotations, add `.Annotations(entproto.Field(N))`.

The 6 `social_media` fields (on Organization, Network, Facility, InternetExchange, Carrier, Campus) get `entproto.Skip()` instead of `entproto.Field(N)`. These use `[]SocialMedia{}` which entproto cannot handle. The skip does NOT consume a field number -- subsequent fields continue the sequential numbering as if social_media was not there.

**C. Add `entproto.Skip()` to every edge.**

Add to existing edge Annotations. Example:
```go
edge.To("campuses", Campus.Type).
    Annotations(
        entrest.WithEagerLoad(true),
        entproto.Skip(),
    ),
```

Run the full generation pipeline in sequence:

1. **Generate proto files from ent schemas:**
   ```
   go generate ./ent/...
   ```
   This runs entc.go which invokes entproto to produce `proto/peeringdb/v1/entpb.proto` containing all 13 message definitions. It also regenerates the existing ent client code, GraphQL schema, and REST handlers (these are pre-existing and should still work).

   Verify the generated entpb.proto file:
   - Contains `package peeringdb.v1;`
   - Contains `message Organization`, `message Network`, etc. for all 13 types
   - Does NOT contain any `social_media` fields (they were skipped)
   - Does NOT contain any edge references
   - Contains `import "google/protobuf/timestamp.proto";` for Time fields
   - Contains `import "google/protobuf/wrappers.proto";` for Optional+Nillable fields

2. **Lint the generated proto files:**
   ```
   go run github.com/bufbuild/buf/cmd/buf@latest lint
   ```
   If lint fails on specific field names (e.g., `ipaddr4`, `ipaddr6`, `rs_asn`), add those specific rules to the `except` list in buf.yaml. Common exceptions needed:
   - `FIELD_LOWER_SNAKE_CASE` if any field names don't match strict snake_case

   Do NOT disable entire lint categories. Only add specific exceptions for fields that entproto generates with names that don't match buf's strict naming rules.

3. **Generate Go code from proto files:**
   ```
   go run github.com/bufbuild/buf/cmd/buf@latest generate
   ```
   This produces:
   - `gen/peeringdb/v1/entpb.pb.go` -- Go types for all 13 proto messages
   - `gen/peeringdb/v1/common.pb.go` -- Go type for SocialMedia message
   - `gen/peeringdb/v1/peeringdbv1connect/entpb.connect.go` -- ConnectRPC handler interfaces
   - `gen/peeringdb/v1/peeringdbv1connect/common.connect.go` -- ConnectRPC handler for common (may be empty if no services)

4. **Verify the generated code compiles:**
   ```
   go build ./gen/...
   ```

5. **Verify the full project still builds:**
   ```
   go build ./...
   ```

6. **Run existing tests to confirm no regressions:**
   ```
   go test ./... -race -count=1
   ```

If `go generate` fails, the most likely causes are:
- Missing entproto.Skip() on an edge (error: `edge "X" does not have an entproto.Field annotation`)
- Missing entproto.Field(N) on a field (error: field without annotation)
- Duplicate field number (error: `field N already defined on message "X"`)
- Unsupported JSON field type (error: `unsupported field type "TypeJSON"`)

Fix any such errors by checking the field numbering table in Task 1's action and ensuring all edges and social_media fields have entproto.Skip().
