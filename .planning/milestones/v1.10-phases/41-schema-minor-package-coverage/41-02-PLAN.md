   - Table-driven test (T-1) with cases:

     - `{"empty bytes", json.RawMessage{}, time.Time{}}` -- empty slice
     - `{"empty object", json.RawMessage(` + "`" + `{}` + "`" + `), time.Time{}}` -- valid JSON but no generated field (generated == 0)
     - `{"invalid json", json.RawMessage(` + "`" + `{invalid` + "`" + `), time.Time{}}` -- Unmarshal error path

4. `TestFetchType_UnmarshalError` -- exercises FetchType lines 247-249:
   - Create httptest.NewServer that serves `GET /api/test` with valid JSON envelope but unmarshalable items:

   - Define a strict type: `type strictItem struct { Name string }` -- but note that Go's json.Unmarshal accepts `12345` for a string field (it converts). Instead use a type with a field that cannot accept the JSON value, e.g., `type strictItem struct { Name struct{} }` where the JSON has `"name": "text"` -- struct{} cannot unmarshal from string.

   - Call `FetchType[strictItem](ctx, client, "test")`
