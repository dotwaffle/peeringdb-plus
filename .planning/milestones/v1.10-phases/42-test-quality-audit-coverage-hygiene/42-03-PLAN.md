For `graph/` resolver errors:
- Use existing resolver_test.go patterns with enttest client
- Test not-found, validation failures, pagination edge cases

For `internal/peeringdb/` client errors:
- Use existing client_test.go patterns with httptest.Server
- Test HTTP errors, malformed JSON, unexpected status codes

For `internal/config/` errors:
- Use existing config_test.go patterns

- Error path tests MUST verify the error content (error message substring or error code), not just `err != nil`
- Add new test cases to EXISTING test files where possible (don't create new files unless no test file exists for the package)
- Do NOT test generated code error paths in ent/ or gen/
