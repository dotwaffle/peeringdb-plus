  <files>files identified in Task 1 audit findings</files>

  <action>
For each test function that FAILED the QUAL-01 audit in Task 1, add at least one meaningful data assertion. The specific fix depends on what the test is doing:

**For tests that only check err == nil on a fetch/query:**
Add assertion on a returned field value. Example:
```go
// Before (weak):
if err != nil { t.Fatalf(...) }
// After (strong):
if err != nil { t.Fatalf(...) }
if result.Name == "" { t.Error("Name is empty, want non-empty") }
```

**For tests that only check HTTP status code:**
Add body content or header assertion. Example:
```go
// Before (weak):
if rec.Code != http.StatusOK { t.Errorf(...) }
// After (strong):
if rec.Code != http.StatusOK { t.Errorf(...) }
body := rec.Body.String()
if !strings.Contains(body, expectedContent) { t.Errorf("body missing %q", expectedContent) }
```

**For tests that check len > 0 without element inspection:**
Add field-level check on first element. Example:
```go
// Before (weak):
if len(results) == 0 { t.Fatal("expected results") }
// After (strong):
if len(results) == 0 { t.Fatal("expected results") }
if results[0].ID == 0 { t.Error("first result has zero ID") }
```

**Rules:**
- Do NOT change test logic or structure -- only ADD assertions
- Use stdlib assertions only (no testify) per project convention
- Keep data assertions simple and deterministic
- For table-driven tests, add the assertion to the common check block, not per-case
- Ensure added assertions reference seeded/expected data, not magic values
- Run `go test -race -count=1` on each modified package after changes

After all fixes, run the full test suite: `go test -race ./...`
