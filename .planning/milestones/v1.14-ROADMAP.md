# Milestone v1.14: Authenticated Sync & Visibility Layer

**Status:** SHIPPED 2026-04-17

**Total Plans:** 21

## Overview

v1.14 delivers the read-path privacy floor. Anonymous callers cannot see `visible="Users"` POC rows on any of the five API surfaces, the sync worker retains full read/write access via a single audited `privacy.DecisionContext` call site, the effective tier is visible to operators at startup / on `/about` / as an OTel span attribute, and production Fly.io now runs authenticated sync by default.

Core unlock: v1.3 wired `PDBPLUS_PEERINGDB_API_KEY` and `Authorization: Api-Key …`, but enabling the key would have written `Users`-visibility POCs to the DB and served them publicly. v1.14 closes that hole before flipping the switch in production.

All 17 requirements (VIS-01..07, SYNC-01..04, OBS-01..03, DOC-01..03) map cleanly to phases 57-62 and resolve to verifiable artifacts. Audit: **PASSED** (see `.planning/milestones/v1.14-MILESTONE-AUDIT.md`).

### Phase 57: Visibility baseline capture

**Goal**: Produce a committed, reviewable empirical baseline showing exactly which fields/rows differ between unauthenticated and authenticated PeeringDB API responses across all 13 types — without which the privacy filter cannot be scoped correctly.

**Depends on**: Nothing (first phase of v1.14)
**Requirements**: VIS-01, VIS-02
**Plans**: 4/4 complete

Plans:

- [x] 57-01-PLAN.md — PII allow-list + pure-function redactor + .gitignore guards (Wave 1)
- [x] 57-02-PLAN.md — Checkpoint + capture loop + FetchRawPage + -capture CLI flag (Wave 2)
- [x] 57-03-PLAN.md — Structural differ + Markdown/JSON emitters + committed-fixture PII guard test (Wave 2)
- [x] 57-04-PLAN.md — Operator-run beta+prod capture + redact + diff commit (Wave 3, autonomous: false)

**Details:**

Empirical visibility baseline captured against `beta.peeringdb.com` for all 13 types in both anon and auth modes, plus a confirmation pass against `www.peeringdb.com` for poc/org/net. PII allow-list + redactor replaces auth-only fields with `<auth-only:T>` placeholders before commit. Structural differ emits per-type Markdown tables + `diff.json`. `TestCommittedFixturesHaveNoPII` guards the committed tree.

Key finding: only two auth-gated surfaces exist — poc row-level visibility and `ixlan.ixf_ixp_member_list_url`. All other 11 types show zero field-level deltas. This drove the Phase 58 decision that no new ent fields are required.

Wall-clock note: walking all 13 types under both auth modes against beta with rate-limit-honouring sleeps takes ≥ 1 hour, intrinsic to the rate ceiling (≤ 20 anon/min, ≤ 40 auth/min). Plan 57-04 was correctly scoped `autonomous: false` — required operator judgement on DIFF.md review.

**Goal**: Install the read-path privacy floor — anonymous queries cannot return rows whose upstream visibility is `Users`-only, while the sync worker retains full read/write access.

- [x] 59-06-PLAN.md — D-15 end-to-end test: all 5 surfaces × both tiers (Wave 6)

**Details:**

`gen.FeaturePrivacy` enabled in `ent/entc.go`; generated `ent/privacy/privacy.go`; `(Poc).Policy()` with NULL-safe predicate (`poc.Or(VisibleEQ("Public"), VisibleIsNil())`) treating NULL as the upstream default (Public) to prevent post-upgrade row floods.

`internal/privctx` foundation (Tier type + WithTier + TierFrom); `internal/config` `PDBPLUS_PUBLIC_TIER` strict-switch parser (default `public`, accepts `users`, fails fast on anything else); `internal/middleware/privacy_tier.go` middleware wired between Readiness and Logging; sync-worker bypass (`privacy.DecisionContext(ctx, privacy.Allow)` as first statement of `Worker.Sync`) with regex-audited single-call-site invariant.

Scope expansion (user-approved): `ent/schematypes` leaf package created to break the schema ↔ ent ↔ schema import cycle that `Policy()` would otherwise trigger (SocialMedia value type moved). Additive, no behaviour change.

Audit hardening: regex upgraded from plan's literal `[^)]*` to `(?s)privacy\.DecisionContext\([^;]*?privacy\.Allow\b` after empirical demonstration that the plan regex missed nested-paren call syntax.

D-15 E2E test covers all 5 surfaces × 2 tiers (19 sub-tests). `TestMiddlewareChain_Order` locks chain position.
